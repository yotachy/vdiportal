# 예측선 불확실성 지형 — 확률 감쇠 레일 + 분위수 팬 + 선 해체

작성 2026-07-21 · 대상 스쿱포지(forge) 4패널 차트 예측 작도
선행 spec: `2026-07-21-prediction-detail-to-distribution-design.md`(근거리 꿈틀 → 원거리 구름)

## 문제

선행 spec으로 예측선에 근거리 꿈틀·원거리 구름이 들어갔으나, **불확실성의 크기 자체는 여전히 전달되지 않는다.**

- 끝점 목표는 확률 50~60%대의 기울어진 동전인데, 화면상 "목표"라는 단정적 마커로 보인다.
- 그 목표까지의 경로 파동은 근거리를 제외하면 확률적으로 의미가 없는데, 선의 굵기·연속성이 동일해 근거리와 원거리가 같은 무게로 읽힌다.
- 현행 `_predLineFade(t)`는 **지평 비율 t의 선형 함수**라 근거가 없다 — 종목·밴드폭과 무관하게 항상 같은 자리에서 흐려진다.

사용자가 인지시키고자 하는 핵심 메시지 2가지:
1. **시간이 갈수록 못 맞힌다** (지평 감쇠)
2. **목표 도달 확률 자체가 낮다** (끝점의 정직한 표기)

## 원칙 / 범위

- **작도(렌더)만 변경. 엔진(forge-core) 불변.** `prediction:{path,lo,hi,counter,anchor,futW,levels,tex}` 값 그대로 사용 → 정확도·백테스트·재현성 완전 무영향.
- **확률 근거는 엔진 산출값만.** 백테스트 파일(`horizon-report.json` 등) 참조 금지 — 외부 파일 의존 0, 종목별로 반응해야 함.
- **웹분석 후에만** 적용. 웹분석 전 은닉(`window._fcPreview`)은 그대로.
- 결정론·난수 없음(꿈틀 시드 규약 유지).
- 세로 스케일(프레이밍) 변화 없음 — `lo`/`hi`가 여전히 최외곽.
- 대상 파일: `forge-draw.js`(작도·범례) · `forge.css`(범례 항목) · 캐시버스터 bump.

## §1 확률의 두 축 분리 (설계의 근간)

실측 확인(demo 7케이스 전부 동일 패턴):

| | k=1 | 중간 | 끝 |
|---|---|---|---|
| raw 방향확률 `_upProb` | 73% | 56% | 46% |
| `calibrateUpProb` 통과 후 | 64% | 60% | 58% |

캘리브레이션(Platt A=0.2117, B=0.3501)은 과신을 교정하는 대신 **값을 55~65% 대로 압축하고 상방 편향을 준다.** 따라서 하나의 숫자로 "정직함"과 "감쇠"를 동시에 표현할 수 없다. 두 축으로 분리한다.

### 숫자로 노출하는 것 — 캘리브레이션된 방향확률 `pCal(k)`

```
rawUp(k) = _upProb(path[k], hi[k], anchor)          // 기존 함수 그대로
calUp(k) = ForgeCore.calibrateUpProb(rawUp(k))
dir(k)   = (path[k] >= anchor) ? +1 : -1            // 봉별 자기 방향(끝점 방향 아님)
pCal(k)  = dir(k) >= 0 ? calUp(k) : (100 - calUp(k))
```

- 용도: 확률 레일 눈금 숫자 · 끝점 라벨. **메시지② 담당.**
- `pCal`이 50% 미만이면 "예측 방향보다 반대가 우세"라는 뜻이며, 이는 숨기지 않고 그대로 표기한다(3차 반대선의 존재 이유와 정합).
- `_upProb`은 현재 `forge-app.js`에 정의돼 있고 `forge-draw.js`에서 이미 호출 중(같은 전역 스코프) — 이동·재정의 없이 그대로 사용.

### 시각 감쇠에 쓰는 것 — 정보량 `z(k)` (확률로 표기하지 않음)

```
z(k) = |log(center[k] / anchor)| / log(hi[k] / center[k])
```

예측 변위가 밴드 반폭 대비 몇 배인가. 위 샘플에서 `0.61 → 0.15 → 0.09` 로 급감 — 압축 없이 감쇠가 선명하다. **메시지① 담당.** 숫자로는 절대 노출하지 않는다(사용자에게 z는 의미 불명).

- `center`는 각 선의 자기 경로(1차 `path` · 2차 `_get2ndPred().path` · 3차 `counter`).
- 분모가 0/음수/비유한이면 z=0으로 폴백(=최저 신뢰).

### 파생 상수

| 상수 | 값(초안) | 의미 |
|---|---|---|
| `Z_LO` | 0.08 | conf=0 하한 |
| `Z_HI` | 0.50 | conf=1 상한 |
| `Z_HORIZON` | 0.25 | 신뢰 지평 임계 |

`conf(k) = clamp01((z(k) − Z_LO) / (Z_HI − Z_LO))`

## §2 작도 구성 요소

```
        ┌ 확률 감쇠 레일(예측구역 상단, 높이 14px)
        ▼
지금│▉▊▋▌▍▎▏▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁
────┼──64%──────60%────╎──────────58%──
    │                  ╎    ░░░░░░░░░░ 68%
    │        ▒▒▒▒▒▒▒▒▒▒╎▒▒▒▒▒▒▒▒▒▒▒▒▒▒ 50%
    │  ╱‾╲_╱‾╲▁▁▁▁▁▁▁▁ ╎ ·  ·  ·   ·   ◆ 42%
    │  실선(conf↑)     ╎  점묘(conf↓)
────┴──────────────────╎──────────────────
                  신뢰 지평 · 이후는 참고
```

### A. 분위수 팬 — 기존 가우시안 밀도 구름을 대체

`sd(k) = log(hi[k] / path[k])` 를 1σ로 취급한다(현행 밴드 ±1σ ≈ 68%, 실측 콘 커버 ~78%와 근사 정합).

- **층 2단**: 50% 구간 = `path[k] × exp(±0.674·sd(k))`, 68% 구간 = 현행 `lo[k]`/`hi[k]` 그대로.
- 구현: 두 개의 폴리곤 fill. ① `lo..hi` 전체를 알파 `0.07 × cloudFade` 로, ② `q50lo..q50hi` 를 알파 `0.16 × cloudFade` 로 덧칠(누적되어 안쪽이 진해짐). 열 단위 그라데이션 아님 — **경계가 뚜렷해야** 크기를 읽을 수 있다.
- 50% 층의 상·하 경계에 알파 `0.35 × cloudFade` 헤어라인(`CW.hair`, 실선).
- 층 전체 알파는 현행 `_predCloudFade(t)` 재사용 → 근거리엔 거의 안 보이고, 선이 해체되는 자리에서 등장.
- 우측 끝(coneR 안쪽)에 `50%` `68%` 소형 라벨. `_evLabel` 회피 레지스트리 사용.
- 3차(반대) 팬은 그리지 않는다 — 1차 팬 + 3차 점묘선으로 충분(과밀 방지). 기존 3차 구름은 제거.
- `_predCloudCol`(세로 가우시안 그라데이션 헬퍼)은 팬으로 대체되어 **삭제**한다.

### B. 확률 감쇠 레일

예측 구역 상단, 높이 `RAIL_H = 14px` 스트립(`padTop` 바로 아래, seamX ~ coneR).

- 봉마다 세로 막대: 높이 = `conf(k) × RAIL_H`, 색 = 방향색(`_rgb1`) 알파 0.5.
- `_hzList(tfUnit(), futW)` 시점(+10/+20/+40/+60 등)에만 `pCal(k)%` 숫자를 표기 — 레일 스트립 **바로 아래** 베이스라인(9px mono, 알파 0.75), 해당 봉 x에 중앙 정렬. 겹치면(직전 라벨과 간격 < 텍스트폭+6px) 뒤엣것 생략.
- 레일 자체가 예측 구역 배경 위에 그려지므로 캔들·y라벨과 겹치지 않음.
- 1차 기준으로만 그린다(2·3차 레일 없음 — 과밀 방지).

### C. 선의 해체 — `_predLineFade(t)` 폐기, `conf(k)` 로 교체

- 현행 `_wigStroke`의 가로 알파 그라데이션은 `t` 기반 10스톱 → **`conf(k)` 기반 봉별 스톱**으로 교체.
- 굵기: 기존 고정 `lw` → `lw × (0.55 + 0.45·conf(k))`. 캔버스 그라데이션으로 굵기는 못 바꾸므로 **세그먼트 단위 스트로크**(봉 사이 구간마다 `beginPath`/`stroke`)로 전환.
- 꿈틀 진폭: `_predWigVal` 내부 `_predLineFade(t)` 호출을 `conf(k)` 인자로 교체(시그니처에 conf 추가).
- **신뢰 지평 이후(k ≥ kHorizon)는 선을 잇지 않고 점묘**: 봉마다 반지름 1.3px 점 1개, 알파 = `0.15 + 0.35·conf(k)`. 1/2/3차 모두 동일 규칙.
- `_predLineFade` / `_PRED_T0` / `_PRED_T1` 은 참조가 사라지면 삭제.

### D. 신뢰 지평선

- `kHorizon` = `z(k) < Z_HORIZON` 을 만족하는 첫 k. 없으면 미표시(끝까지 신뢰 유지되는 종목).
- 차트 전 높이(padTop ~ ch−padBot) 세로 점선(`CDASH.fine`, 알파 0.3) + 상단에 `신뢰 지평` 배지.
- 배지 우측 소형 회색 텍스트 `이후는 참고`.
- 1차 경로 기준으로만 산출.

### E. 끝점 라벨 개정

`_predEndDeco`의 label 인자를 변경한다(함수 자체는 불변).

- 1차: `"1차·" + _p1disp + "%"` (raw 가중평균) → `"1차·" + pCal(futW−1) + "%"`.
- 2차·3차도 동일하게 각자의 `pCal(끝봉)` 사용.
- `pCal < 50`이면 라벨 색을 회색(`#8a92b2`)으로 강등하고, `_epicenterMark` 알파를 낮춘다 → 목표 마커가 시각적으로 약해짐.
- 툴팁은 캔버스라 불가 → 범례 `p1` 항목 툴팁에 "50% 미만이면 반대 시나리오가 우세하다는 뜻" 문구 추가.

### F. 범례

- `_predVis = { band, fan, rail, p1, p2, p3 }` (기본 전부 true). `band`는 lo/hi 경계 헤어라인, `fan`은 분위수 층, `rail`은 확률 레일.
- `_renderChartLegend` 항목 추가: `분위수 팬` (스와치 sq, 방향색) · `확률 레일` (스와치 sq, 방향색 흐림).
- 각 항목 툴팁:
  - 분위수 팬 — "예측이 이 안에 들어올 확률. 안쪽 진한 층 50%, 바깥 경계 68%. 뒤로 갈수록 벌어지는 폭이 불확실성입니다."
  - 확률 레일 — "각 시점의 예측 신뢰도. 막대가 낮아질수록 그 시점 예측은 근거가 약합니다. 숫자는 캘리브레이션된 방향 적중 확률."
  - 1차 종합지표(기존 문구에 추가) — "끝점 확률이 50% 미만이면 반대 시나리오가 우세하다는 뜻입니다."

## 데이터 흐름

`fcDrawMainChart(series, pred)` → 예측 블록(`else(!_prev)`):

1. `_predZ(center, hi, anchor)` → `z[]` 배열, `conf[]` 배열, `kHorizon` 산출 (1차 기준)
2. `if(_predVis.band)` lo/hi 경계 헤어라인 (현행 유지)
3. `if(_predVis.fan)` 분위수 팬 2층 + 우측 `50%`/`68%` 라벨
4. `if(_predVis.rail)` 확률 감쇠 레일 + `pCal` 눈금 숫자
5. `kHorizon` 있으면 신뢰 지평선 + 배지
6. `if(_predVis.p3)` 3차 점묘/세그먼트선 + 끝점 라벨(`pCal`), `p1` 동일
7. (2차는 `drawEvidence` 경로 — 같은 conf/점묘 규칙 적용)
8. `_renderChartLegend(_pd)`

## 엣지 / 에러

- `hi[k] <= path[k]` 또는 비유한 → `z=0`(conf=0) 폴백. 해당 봉은 점묘로.
- `futW < 4` → 레일·팬은 그려지되 눈금 숫자는 `_hzList`가 걸러 1~2개만.
- `kHorizon === 0`(첫 봉부터 무신뢰) → 지평선을 seamX에 겹치지 않게 최소 `k>=1`부터 탐색.
- `kHorizon`이 없음(끝까지 z ≥ Z_HORIZON) → 지평선·"이후는 참고" 미표시, 점묘 구간 없음.
- 2차 없음(`_get2ndPred` null 또는 1차와 동일) → 스킵(현행 로직 유지).
- 밴드가 극히 좁음 → z가 매우 커져 conf=1 고정 → 전 구간 실선. 정상 동작(고신뢰 종목).
- 다크/라이트 테마: 팬·레일 색은 방향색(bull/bear, 테마 무관 상수) + 알파만 사용 → 테마 leak 없음.

`forge-core.test.js`는 엔진 전용이므로 **이번 변경으로 케이스가 늘지 않는다**(엔진 불변 = 246/246 유지 확인만).

작도 로직은 순수 함수 부분을 실소스 추출로 검증한다(기존 관례):

1. `_predZ` — 알려진 path/hi/anchor로 z 값 검증, 지평 증가 시 단조 감소 확인.
2. `conf` — Z_LO 이하 → 0, Z_HI 이상 → 1, 중간 선형.
3. `kHorizon` — 임계 교차 지점 정확도, 교차 없음 → null, k=0 케이스 → 최소 1.
4. `pCal` — 상승 예측/하락 예측 각각 방향 반영, 50% 미만 케이스 발생 확인.
5. 분위수 층 가격 — `sd=log(hi/path)` 로 `±0.674σ` 재구성이 `lo<q50lo<path<q50hi<hi` 순서 만족.
6. 회귀: `_predVis` 전 항목 off → 예측 영역에 아무것도 안 그려짐(에러 없음).

헤드리스 시각 검증(WSL playwright, [[headless-verify-wsl]]): 실티커 1종 웹분석 후 스크린샷 — 레일·팬·지평선·점묘 존재, 라벨 겹침 없음. **읽기 전용 함수만 사용**([[headless-live-tests-readonly]]).

## 구현 중 설계 정정 (2026-07-21, 헤드리스 시각 검증에서 발견)

정직 기록 원칙에 따라 실패한 설계를 남긴다.

### ① 정보량 z 는 감쇠 구동값으로 실패했다 — 밴드 상대확장으로 교체

§1의 `z = |log(center/anchor)| / log(hi/center)` 는 **횡보 예측에서만 감쇠한다.** 드리프트는 k에 선형, 밴드는 √k로 벌어지므로 **추세 예측일수록 z가 커진다.**

| 국면 | k=1 | k=6 | k=13 | k=24 |
|---|---|---|---|---|
| 횡보 | conf 1.00 | 1.00 | 0.18 | 0.06 (정상) |
| 완만추세 | 1.00 | 1.00 | 1.00 | 1.00 (감쇠 없음) |
| 강추세 | 1.00 | 1.00 | 1.00 | 1.00 (z 1.4→3.9 증가) |

과신이 가장 위험한 추세 예측에서 선이 끝까지 굵은 실선으로 남고 지평선도 안 떴다. 최초 설계 검토 때 이 위험을 인지하고 검증했으나, 하필 표본이 횡보 예측이라 통과했다. **작도 변경은 국면별로 검증해야 한다.**

교체된 정의:

```
W(k)    = log(hi[k] / lo[k])                          // 밴드 로그폭 — k에 대해 단조 증가
conf(k) = 1 − (W(k) − W(0)) / (W(n−1) − W(0))         // 총 확장분 대비 진행도의 역
신뢰 지평 = conf(k) < 0.5 인 첫 k (k ≥ 1). 없으면 null
```

- 밴드는 반드시 벌어지므로 **단조 감소가 수학적으로 보장**된다. 국면 무관.
- 실측 실선 구간: 횡보 21% / 완만추세 25% / 강추세 33% (라이브 60봉 케이스 33%).
- **`W(0)` 비율(`sqrt(W0/W(k))`)은 쓰지 않는다** — 엔진이 콘을 seam에서 인위적으로 좁게 시작시켜(`res*(0.15+0.03k)`) `W(0)`이 왜곡돼 있고, 라이브에서 60봉 중 4봉만 실선으로 남는 과감쇠가 났다.
- conf 는 이제 `lo`/`hi` 만으로 결정된다 — `center`·`anchor` 를 쓰지 않는다.

### ② 레일 눈금 숫자 제거 + 하단 이동

§2 B의 시점별 `pCal` 숫자는 **엔진 콘이 드리프트 대비 좁아 saturate 된다.** 라이브 실측에서 `hi`가 `path`의 +0.8% 수준이라 `_upProb` 가 100으로 포화 → Platt 상한 86에 고정 → +10/+20/+40/+60일 **전부 `86%`**. 같은 숫자가 네 번 찍히며 메시지②(도달 확률이 낮다)와 정반대로 과신을 키웠다.

- 레일에서 숫자를 제거하고 **막대(시각 감쇠)만** 남긴다. 확률 표기는 끝점 라벨 하나로 일원화.
- 레일 위치를 **차트 하단(x축 바로 위)** 으로 이동 — 상단은 좌상단 범례 박스(`#fcLegend`, 캔버스 위 DOM 오버레이)에 가려 거의 안 보였다. 신뢰 지평 배지도 같은 이유로 하단 레일 위로 이동.
- 범례 항목명 `확률 레일` → `신뢰 레일`.

### ③ 신뢰 지평선은 예측선 가시성에 종속

범례 6항목을 모두 끄면 세로 점선과 배지만 남았다. 지평선은 "예측선이 여기서 해체된다"는 표시이므로 `_predVis.p1 || p2 || p3` 일 때만 그린다.

### ④ 팬 라벨은 `_evLabel` 회피 레지스트리를 쓰지 않는다

§2 A는 `50%`/`68%` 라벨에 `_evLabel` 겹침 회피를 쓴다고 했으나, `_evLabelBoxes` 는 이후 `drawEvidence` 에서 리셋되므로 여기서 등록해도 무의미하다. 대신 붐비는 우측 끝(끝점 라벨·가격 pill)을 피해 **콘 span의 72% 지점**에 직접 그린다. 콘이 좁아 두 라벨의 세로 간격이 11px 미만이면 바깥 `68%` 를 생략한다.

같은 성격의 소소한 편차: §2 E의 "`_epicenterMark` 알파를 낮춘다" 는 실제 시그니처 5번째 인자가 알파가 아니라 **scale** 이라 `scale 0.6` 으로 구현됐다.
