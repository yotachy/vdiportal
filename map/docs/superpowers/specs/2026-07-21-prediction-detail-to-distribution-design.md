# 예측선: 근거리 디테일 → 원거리 확률분포 (+ 범례 토글)

작성 2026-07-21 · 대상 스쿱포지(forge) 4패널 차트 예측 작도

## 문제

현재 1/2/3차 예측선은 현재가에서 목표까지 **매끈한 단조 곡선**으로 그려진다. 실제 가격은 그렇게 움직이지 않아 비현실적이고 납득이 안 간다. 사용자 제안: **앞부분(근거리)일수록 디테일한 차트 움직임으로, 뒤로 갈수록 확률분포식으로** 그리면 현실적이고 설득력 있다. 1/2/3차 기준은 유지. 데이터는 많을수록 좋고, 대신 범례 클릭으로 요소를 숨길 수 있게.

## 원칙 / 범위

- **작도(렌더)만 변경. 엔진(forge-core) 불변.** `prediction:{path,lo,hi,counter,anchor,futW}` 값 그대로 사용 → 정확도·백테스트·재현성 완전 무영향.
- **웹분석 후에만** 적용. 웹분석 전 은닉(`window._fcPreview`)은 그대로 — 이 변경은 `fcDrawMainChart`의 `else(!_prev)` 예측 작도 블록에 국한.
- **결정론·재현 가능·난수 없음.** 근거리 꿈틀은 종목 데이터에서 시드된 결정론적 함수 → 같은 종목·기간 = 같은 그림(`Math.random` 금지, 엔진의 잡음 없는 원칙을 작도층에서도 준수).
- 대상 파일: `forge-draw.js`(작도·범례) · `forge.css`(범례 토글 스타일) · 필요 시 `forge.html`(없음 예상). 캐시버스터 bump.

## 구성 요소

### A. 근거리 꿈틀 3선 — **계산된 반등/하락 (S/R 반응 + AR 결)** ★ v2(2026-07-21)

각 예측선(1차 `path` · 2차 `_get2ndPred().path` · 3차 `counter`)을 매끈 선 대신 **의미 있는 미세움직임**으로. 꺾이는 지점이 무작위가 아니라 **실제 기술적 레벨**에 위치.

> **v1(폐기)**: 정현파 합/순수 랜덤워크 = 무의미한 시각 질감. 사용자 지시 "반등/하락 지점을 무조건 계산"으로 v2 대체.

- **엔진 노출(순수 추가·로직 불변)**: `prediction`에 두 필드 추가 — 값 계산은 이미 하던 것(counterTarget용 `_lvls`, path에 반영된 `_texArr`)을 **surface만** 함. 예측·정확도·백테스트 **완전 동일**(246/246 회귀 확인).
  - `levels`: S/R 가격 배열 = `_lvls`(피보·구조 스윙·매물대 POC/VAH/VAL·볼린저·일목 구름·VWAP). 정렬·근접 중복 제거.
  - `tex`: AR2 질감 배열 = `_texArr`(최근 로그수익률 단기 자기상관 투영, 없으면 null).
- **작도 헬퍼 `_predWigSeqSR(n, center, lo, hi, levels, tex, seed)`** → 정규화 시퀀스 `[-1,1]`:
  1. **AR 결(기본 미세결)**: `tex[k]`가 있으면 그대로(종목 실제 되돌림 성향 — 잦으면 톱니, 추세적이면 매끈). 없으면 최소 시드 결(`_mulberry32` 저진폭)로 폴백해 완전 평탄 방지.
  2. **S/R 반응(꺾임 지점)**: 각 k에서 `center[k]` 근처 유의 레벨을 탐색(reach = 국소 밴드 반폭 × `SR_REACH`). 있으면 **자석 변위** `pull = (levelPrice − center[k]) × magnet(dist, sig)` 가산 — 근접할수록 강하게 끌려 레벨에서 멈칫·반등. 여러 레벨은 최근접 우선(가중합 상한). → 꺾이는 극값이 레벨에 배치.
  3. 합성 `raw[k] = SR_W·pull_norm[k] + AR_W·ar[k]` → 전체 max로 [-1,1] 정규화.
- **적용(`_predWigVal` 재사용)**: y = center + `amp(k)·seq[k]·lineFade(t)`, `amp = 0.5×국소밴드반폭`, 밴드 `[lo,hi]` 하드 클램프. center(=예측)·band 값 불변, 시각 변위만 가산.
- 1/2/3차: `levels`·`tex`·`series`는 공유, 각 선의 `center`(path/2nd/counter) 기준으로 반응·결 → 3선이 각기 다르되 같은 레벨에서 반응(정합).
- 명명 상수: `SR_REACH`(레벨 탐색 반경)·`SR_W`/`AR_W`(합성 비중)·`magnet` 강도 — 조정 용이.
- 스타일: 1차 실선(방향색)·2차 파랑 점선·3차 반대 점선 — **현행 색/대시·크로스페이드·구름 유지**.

### B. 크로스페이드 (하드 경계 없음)

지평 위치 `t = k/futW ∈ [0,1]`에 대해:
- `lineFade(t)` = clamp01(`1 - (t - T0)/(T1 - T0)`) — `T0≈0.15`(이때까지 선 100%)에서 `T1≈0.75`(이후 선 0%)로 매끄럽게 감소.
- `cloudFade(t)` = clamp01(`(t - C0)/(C1 - C0)`) — `C0≈0.10`에서 `C1≈0.70`로 증가. (선이 사라지는 자리를 구름이 채움 — 두 페이드 구간이 겹쳐 자연스러운 인계.)
- 상수 T0/T1/C0/C1는 파일 상단 근처 명명 상수로(추후 조정 용이).

### C. 원거리 밀도 그라데이션 구름 (2개)

- **1차 구름**: 각 x열(크로스페이드+원거리 구간)에서 세로 방향 밀도 — **중앙=`path[k]`에서 진하고, `lo[k]`/`hi[k]` 가장자리로 가우시안형 감쇠**. 색=방향색조 저알파, 전체 알파 `∝ cloudFade(t)`.
- **3차(반대) 구름**: 같은 방식으로 `counter[k]` 중심 + 반대편 확률 퍼짐(하방/상방). 1차보다 옅게(반대 시나리오 = 보조).
- 구현: 원거리 각 x열마다 세로 `createLinearGradient(lo..hi)` + 중앙 피크 알파 스톱 → `fillRect(x, ...)`. 열 폭 = 예측 봉 간격. (성능: futW≤60 × ~2 구름 = 저비용.)
- 밴드 경계선(lo/hi 헤어라인)은 유지(구름의 외곽 = 기술적 최대범위).

### D. 범례 클릭 토글 (신규)

`#fcLegend`(`_renderChartLegend`)의 4항목을 on/off 스위치로.

- 상태: `_predVis = { band:true, p1:true, p2:true, p3:true }` (세션 인메모리, 기본 전부 켜짐).
  - `band` = lo/hi 밴드 경계 + 밀도 구름(1차+3차) 전체
  - `p1`/`p2`/`p3` = 각 꿈틀 예측선
- 렌더: 각 `.fc-leg-item`에 `data-predkey` + 클릭 핸들러. off면 `.off` 클래스(딤/취소선·스와치 흐림).
- 클릭 → 해당 키 토글 → 예측 재작도(`fcRenderForecast(lastResult.prediction)` 또는 `renderChart` 재호출)로 반영.
- 작도 블록은 `_predVis`를 읽어 각 요소 그리기 분기.
- 드래그 이동(`.fc-leg-grip`)·툴팁(`.fc-leg-tip`)은 불변. 지표 32종 범례(`_evVisible`/`_focusInd`)와는 **완전 별개 상태**(혼동 없음).
- 항목 라벨은 요소 반영: `밴드·구름` / `1차 종합` / `2차 선택` / `3차 반대`.

## 데이터 흐름

`fcDrawMainChart(series, pred)` → (기존) 캔들 → **예측 블록(else)**:
1. seed·crossfade 상수 계산
2. `if(_predVis.band)` 밀도 구름(1차·3차) + lo/hi 경계
3. `if(_predVis.p1)` 1차 꿈틀선, `p2`/`p3` 동일
4. 끝점 목표·확률 라벨(`_predEndDeco` 등)·"방향 참고" 표기 — 현행 유지(각 라인 토글에 종속)
5. `_renderChartLegend`가 `_predVis` 상태로 항목 딤 반영

## 엣지 / 에러

- `path`/`counter`/`lo`/`hi` 길이 불일치·빈 배열 → 해당 요소 스킵(기존 가드 유지).
- `futW` 작음(<4) → 크로스페이드 여전히 동작(구간 축소), 근거리만 보일 수 있음.
- 2차 없음(`_get2ndPred` null·1차와 동일) → 2차 스킵(현행 로직 유지).
- 밴드가 매우 좁음 → `amp` 상한이 0에 수렴 → 꿈틀 거의 없음(과장 방지 정상 동작).

## 테스트

- **결정론**: 같은 `pred`·`series` 2회 → `wig(k)` 좌표열 동일(시드·레벨·tex 재현). 실소스 추출 단위검증.
- **밴드 상한**: 모든 k·시드에서 꿈틀값이 `[lo,hi]` 안(하드 클램프).
- **크로스페이드 단조성**: `lineFade` 감소·`cloudFade` 증가·둘 다 [0,1].
- **★ 꺾임=레벨 근접**: 합성 시퀀스의 국소 극값(반등/하락 지점)이 실제 `levels` 근처(중앙값이 레벨에 근접한 k)와 정렬됨 — **레벨 있는 케이스가 레벨 없는(폴백) 케이스보다 극값-레벨 근접도 유의 높음**을 수치 검증.
- **★ AR 데이터 반영**: 평균회귀 합성 수익률 → 되돌림(부호변화) 잦음 vs 추세 수익률 → 매끈함, 결이 tex를 따름.
- **토글**: `_predVis` 각 키 off → 해당 그리기 분기 스킵(플래그 기반 단위검증).
- **엔진 추가 안전**: `levels`/`tex` 노출은 순수 추가 → forge-core 246/246 **불변**(예측·정확도·백테스트 동일).
- **헤드리스 시각**: 실 티커 웹분석 후 예측선이 S/R 레벨에서 반응·근거리 결·원거리 구름·범례 딤 육안 확인(가능 범위).

## 비목표(YAGNI)

- 엔진 path/band 값 변경(정확도 개선) — 안 함(작도 전용).
- `_predVis` 서버/로컬 영속 — 이번엔 세션 인메모리만(추후 필요 시).
- 캔들형 예측 투영 — 사용자 선택은 꿈틀 3선(캔들 아님).
