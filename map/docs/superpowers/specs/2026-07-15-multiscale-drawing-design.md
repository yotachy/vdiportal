# 설계: 데이터기반 다중스케일 작도 + 중요도 위계 (작도/시각 전용)

> 대상: `map/forge-core.js`(순수 앵커/스코어 헬퍼) · `map/forge-draw.js`(작도·시각 위계) · `map/forge-core.test.js`(헬퍼 단위테스트). 배포: git + cafe24 `www/map/`.
> 작성: 2026-07-15. 브레인스토밍 합의 기반. **[[scoopforge-multiscale-drawing]]** 메모리와 정합.

## 배경 / 요구 (사용자 지시)

- **고정 기준(무 자르듯) 금지**: degree를 봉 개수(120/60/24)로 못 박지 말 것. **데이터에서 유의미한 스윙/구간을 스케일 전반에 걸쳐 발굴**(최대치=멀리서 보이는 큰 구조 ~ 최소치=최근 유의미 구간). 봉 간격 기준으로 자연스럽게.
- **애널리스트처럼 폭넓게**: "다중"조차 3개만 뽑는 게 아니라 사람이 차트 보듯 **후보를 넉넉히** 작도.
- **중요도 위계**: 작도가 많아도 아주 중요한 지점·구간·지지선을 **눈에 띄게(굵게·라벨·설명 부착 가능)**, 나머지는 **흐리게(디밍)**. "넉넉히 그린다 → 중요도 순위 → 강조/디밍".
- **현행 문제**: Gann이 단일 앵커(`_domSwing` 하나)에서만 작도 = 시야 좁음. (trend·fib는 이미 장/중/단 다중.)

## 범위 / 불가침 제약

- **작도/시각만 개선. 엔진(bias/예측/run) 절대 미변경** — 사용자 결정: 엔진 다중스케일화는 "나중에 분석엔진 개선 때 꼭" 별도 진행(백테스트·자명규칙·OOS 검증 규율 필요). 지금은 `analyzeX.bias`·`run()`·`_drifts`·회고 인프라 **한 줄도 안 건드림** → 백테스트/회고 베이스라인 불변, 검증 불필요.
- 신규 헬퍼(`collectAnchors`·significance)는 **순수 함수**로 forge-core에 두되 **`run()`에 배선하지 않음**(엔진 미연결). 단위테스트만.
- 좌측 세로 accent line 금지. 전역 스코프·로드순서 고정. 서버 데이터 파일 불가침.

## 설계 A — 공용 앵커 발굴 (`collectAnchors`, 순수)

`collectAnchors(price, opts) → [anchor]` — 고정 창 없이 데이터가 degree를 정한다.

- **민감도 사다리**: `detectSwings(price, s)`를 `s ∈ [0.18, 0.12, 0.08, 0.05, 0.035, 0.02]`(굵게→가늘게)로 실행. 굵은 s = 큰 스윙(소수·장기), 가는 s = 작은 스윙(다수·단기).
- 각 스윙 레그(연속 두 피벗 from→to) = 앵커 후보 `{ fromIdx, fromPrice, toIdx, toPrice, dir, degree }`. `degree` = 그 레그가 살아남는 **가장 굵은 s의 사다리 인덱스**(작을수록 큼).
- **dedup**: 같은 피벗쌍이 여러 s에서 나오면 가장 굵은 degree로 1건 유지(idx 근접 병합).
- 반환은 significance 내림차순 정렬(설계 B).

## 설계 B — 중요도 스코어 (`anchorScore`, 순수)

각 앵커에 `significance ∈ [0,1]` + `reason`(설명 훅):
- **크기(degree/magnitude)**: 굵은 사다리 + `|toPrice−fromPrice|/range` 클수록↑ (가중 0.35)
- **최근성**: `toIdx`가 최근일수록↑ (0.2)
- **터치/반응**: 그 레그의 레벨(또는 1×1 선) 근방에서 가격이 되돌려진 횟수(스윙 피벗이 근접) (0.25)
- **컨플루언스**: 다른 앵커/레벨과 겹치는 가격대 가점 (0.1)
- **현재가 근접도**: 앵커·투영선이 현재가에 가까울수록↑ (0.1)
- `reason`: 스코어 상위 요인을 문자열로(예: "대형 스윙 · 3회 반응") — 라벨/툴팁·후속 설명에 사용.

## 설계 C — 시각 위계 (강조/디밍)

`_drawGannLayers`(및 후속 작도)가 significance→시각 매핑:
- **넉넉히 다 그림** — 상위 K개(예: ≤6) 앵커. 하위는 옅게.
- **불투명도·굵기·라벨**: 상위 1~2 = 굵은 골드 + 라벨(reason), 중위 = 중간, 하위 = 흐린 얇은 선.
- **밀도도 significance가 결정**: 굵은(중요) 앵커는 전체 7각 부챗살, 흐린 앵커는 1×1 위주(고정 규칙 아님, 점수 기반).
- **설명 부착 훅**: 상위 앵커 라벨에 `reason` 표시(나중에 상세 설명 확장 여지).

## 설계 D — Gann 첫 적용

- `analyzeGann`은 **엔진 bias 계산은 그대로**(단일 지배 스윙, `run()` 무변경) 두고, **작도용 필드 `anchors: [ {…, significance, reason, angles} ]`를 추가 반환**(fib가 `degrees`를 반환하는 선례와 동일 — 드로우 페이로드 확장, bias 불변).
  - `anchors`는 `collectAnchors`+`anchorScore` 상위 K개. 각 앵커의 1×1 = 그 스윙 실기울기(앵커→정점), 2×1~1×4 부챗살.
- `_drawGannLayers`가 단일 앵커 대신 `pt.anchors`(또는 `gn.anchors`)를 순회하며 설계 C 위계로 작도.
- **가시 윈도우 클램프**: 각 부챗살 선을, 앵커가 화면 좌단 밖이면 **좌단 진입점(fiMin 봉의 투영가)부터** 그림 → `fiToX(aIdx)=NaN` 무음 미표시 버그 동시 해소.

## 설계 E — 확장성

- `collectAnchors`·`anchorScore`·시각위계 헬퍼는 **공용**. 이후 추세선·fib·S/R·structure 작도가 재사용, **엔진(나중)**도 동일 구조 소비 가능.
- 이번엔 **Gann 작도만** 실제 배선(증명). 나머지는 문서화된 확장 여지.

## 엔진 통합 체크리스트 — **해당 없음(의도적)**

- `run()`·`_drifts`·`evalBlocks`·`analyzeX.bias`·`seedDefaultStrategy`·`map/backtest/` **미변경**. 이 스펙은 작도 페이로드/작도함수/공용 헬퍼만.

## 검증

- `node --test map/forge-core.test.js` — `collectAnchors`(사다리별 스윙 수확·dedup·degree)·`anchorScore`(요소별 단조성) 단위테스트 추가. 기존 233 그린 유지, **엔진 예측 무변화 회귀**(gann 노드 예측 target 불변 — bias 미변경 확인).
- 헤드리스([[headless-verify-wsl]]): 실데이터 형태(느린 드리프트+진동)로 **여러 앵커 부챗살이 스케일별로 그려지고 상위만 굵게·나머지 흐리게**, 창밖 앵커도 좌단부터 렌더 확인. 라이브 쓰기함수 금지([[headless-live-tests-readonly]]).
- 배포 후 curl 200 + 반영.

## 순서 (과제 = 재개 가능)

1. 공용 헬퍼 `collectAnchors`+`anchorScore` (순수·TDD).
2. `analyzeGann`에 작도용 `anchors` 필드 추가(bias 불변).
3. `_drawGannLayers` 다중앵커 + 시각위계 + 윈도우 클램프.
4. 검증(테스트·헤드리스) + 배포 + 스코어카드 기록.
- 각 단계는 SDD 원장에 기록 → **세션 종료돼도 원장+git log로 이어서 재개**.

## 열린 항목 / 향후

- 확장: 추세선·fib·S/R·structure 작도도 `collectAnchors` 위계 적용.
- **엔진 다중스케일화**(bias/예측) — 별도 과제, 백테스트 검증 동반(사용자: "나중에 분석엔진 개선 때 꼭").
- 상위 앵커 `reason`에 상세 설명(툴팁/패널) 확장.
