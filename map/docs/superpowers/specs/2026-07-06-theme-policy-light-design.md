# 색 테마 정책 통일 + 밝은 테마 — 설계

- 날짜: 2026-07-06
- 대상: `map/forge.html` (UI + 차트 캔버스). `forge-core.js`·데이터 파일 미변경.
- 목적: 테마 전환 시 **일부 토큰만 바뀌어 생기는 불일치**를 없애 정책을 통일하고, 중복 다크 테마를 정리하며, **밝은 테마(2종)**를 추가한다.

## 배경(현행 문제)

`THEMES`(forge.html ~L7961) 7종(navy·midnight·charcoal·teal·cement·purple·orange)은 각 테마가 **6개 변수만** 오버라이드한다: `--bg --panel --surface --raised --raised2 --gold`.

`:root`의 나머지 테마 관련 토큰은 **어느 테마도 안 바꾼다** → 항상 기본 다크값 고정:
- 텍스트: `--ink`(주) `--eth` `--muted` `--faint`
- 테두리: `--line` `--edge`
- 액센트: `--gold-dim`

결과: ① 액센트 테마(teal/purple/orange)는 `--gold`만 바뀌고 `--gold-dim` 테두리·기타 골드 톤은 그대로 → **부분 불일치**. ② 텍스트/테두리가 안 바뀌므로 **밝은 테마가 불가능**(배경만 밝히면 밝은 텍스트가 남아 판독 불가). 또한 UI 곳곳이 토큰 밖 하드코딩 색을 씀: `rgba(255,255,255,…)` 호버 **40곳**, `rgba(11,15,20,…)` 다크 오버레이 **26곳**, 서브패널 그리드 `#2b3647`/`#2a3346`, 차트 캔버스 `FC_GOLD/FC_BULL/FC_BEAR/FC_GRID` + 인라인 hex.

## 결정 사항(브레인스토밍 합의)

- **전체 토큰 일괄**: 모든 테마가 완전한 토큰 세트를 지정.
- **정리**: 그레이 2종(charcoal·cement) 삭제, midnight 유지 → 다크 5종.
- **밝은 테마 둘 다**: 페이퍼(밝은 UI + 다크 차트) + 데이라이트(완전 밝은, 차트 포함).

## 1) 통일 토큰 계약

테마가 제어하는 **완전한 세트**(모든 테마가 전부 지정). UI 크롬은 이 토큰만 사용한다.

| 그룹 | 토큰 | 설명 |
|---|---|---|
| 표면 | `--bg` `--panel` `--surface` `--raised` `--raised2` | 페이지·패널·카드·상승 표면 |
| 테두리 | `--line` `--edge` | 기본/강조 테두리 |
| 텍스트 | `--ink` `--eth` `--muted` `--faint` | 주·보조·약·최약 |
| 액센트 | `--gold` `--gold-dim` | 액센트·감쇠 액센트 |
| **신규(토큰화)** | `--hover` `--scrim` `--grid` | 호버 오버레이 / 모달·오버레이 딤 / 서브패널·차트 격자선 |
| 상수(테마 무관) | `--bull`#46c28e · `--bear`#e06a6a · `EV_COLORS`(30 지표 정체성) · `--r-*` | 시맨틱·정체성·라운드 |

- **신규 토큰 3종**(`--hover`·`--scrim`·`--grid`)을 `:root`에 추가하고, 하드코딩 40+26+격자를 이들로 치환한다. 다크 기본값: `--hover:rgba(255,255,255,.05)` `--scrim:rgba(11,15,20,.72)` `--grid:#1b2334`. 라이트: `--hover:rgba(0,0,0,.045)` `--scrim:rgba(30,34,42,.5)` `--grid:#d7dde6`.
- `applyTheme`는 세트 전체를 `documentElement.style`에 setProperty. 테마 객체 하나가 **모든 토큰**을 담는다(누락 시 기본값 폴백은 두지 않음 — 완전성 강제).

## 2) 테마 목록(정리 후 · 7종)

**다크(5)**: `navy`(네이비·기본) · `midnight`(미드나잇) · `teal`(딥틸) · `purple`(로열퍼플) · `orange`(앰버오렌지). — `charcoal`·`cement` 삭제.

**라이트(2)**:
- `paper`(페이퍼) — 밝은 UI + **다크 차트**. bg `#f4f2ec`(웜 페이퍼)·panel `#ffffff`·ink `#1c2128`·line `#e2e0d8`·gold `#b0842f`(라이트에서 대비 확보한 골드)·hover `rgba(0,0,0,.045)`. 차트 캔버스는 다크 유지(아래 3참조).
- `daylight`(데이라이트) — **완전 밝은**(차트 포함). paper 토큰 + 차트 캔버스도 밝게.

테마 팝업은 "다크"/"라이트" 서브헤더로 그룹 표시(`THEMES` 항목에 `group:"dark"|"light"` 필드 추가, `renderThemePop`이 그룹별 렌더).

## 3) 밝은 테마 · 차트 처리

차트 캔버스(`#fcMain`·오실레이터 서브패널·콘/예측선)는 `FC_*` 상수·인라인 hex·격자로 그린다.

- **페이퍼**: 차트 캔버스 배경을 **다크 고정**(데이터 가독성). 즉 hero/차트 영역만 다크 톤 유지 → 별도 `--chart-bg` 토큰을 두고 paper에선 다크값. UI 크롬(패널/텍스트/레일/모달)은 밝음. **차트 FC_* 재튜닝 불필요**.
- **데이라이트**: `--chart-bg` 밝음 + **`FC_*`·격자·예측선·밴드·서브패널 그리드를 테마 인지로**. 방법: `FC_GOLD`→`--gold`, `FC_GRID`→`--grid`, 캔버스 배경→`--chart-bg`, 서브패널 밴드 배경/격자→토큰. 캔버스에서 CSS 변수 읽기: `getComputedStyle(document.documentElement).getPropertyValue('--gold')` 등을 렌더 시 캐시(`_themeColors()` 헬퍼). `FC_BULL/FC_BEAR`(가격 방향)는 상수 유지.

`renderTheme`/`renderHeroZoom`가 테마 변경 시 차트 재드로하도록 `applyTheme`에서 호출(현재 renderHeroZoom만 호출 → renderChart/서브패널 갱신 추가).

## 4) 하드코딩 색 토큰화(통일성 핵심)

| 대상 | 현행 | 치환 |
|---|---|---|
| 호버(40곳) | `rgba(255,255,255,.03~.07)` | `var(--hover)` (강도 단계가 필요하면 `--hover`/`--hover-2` 2단계) |
| 오버레이/딤(26곳) | `rgba(11,15,20,.6~.95)` | `var(--scrim)` 또는 반투명 패널은 `--panel` 기반 |
| 서브패널 격자 | `#2b3647` `#2a3346` `#1b2334` | `var(--grid)` |
| 차트 골드/격자(데이라이트) | `FC_GOLD` `FC_GRID` 등 | `_themeColors()` 경유 토큰 |

- bull/bear·EV_COLORS(지표별 색)는 라이트에서도 시인성 OK(중간 채도) → 상수 유지. 필요 시 라이트에서 살짝 어둡게 하는 미세조정은 범위 밖(YAGNI).

## 구현 단계(phasing)

1. **토큰 계약 확정**: `:root`에 `--hover --scrim --grid --chart-bg` 추가(다크 기본값). 완전 토큰 세트 목록 확정.
2. **DOM 하드코딩 토큰화**: hover 40 → `--hover`, 오버레이 26 → `--scrim`/`--panel`, 격자 → `--grid`. (다크 무회귀 — 기본값이 현행과 동일)
3. **테마 세트 전면화**: navy·midnight·teal·purple·orange를 완전 토큰 세트로 확장(charcoal·cement 삭제).
4. **페이퍼 추가**: 라이트 UI 토큰 세트 + `--chart-bg` 다크. 팝업 그룹 서브헤더.
5. **차트 FC_* 토큰화**: `_themeColors()` 헬퍼, FC_GOLD/GRID·캔버스 배경·서브패널 격자를 토큰 경유. 데이라이트에서 검증.
6. **데이라이트 추가**: 완전 밝은 세트(`--chart-bg` 밝음).

각 단계 헤드리스 스크린샷으로 다크 무회귀 + 라이트 판독성 검증.
