# 캔버스 단순화 (Scoop Forge) — 설계

**날짜:** 2026-07-01
**대상:** `map/forge.html` (보드 캔버스 상호작용 · 노드 편집 UX)
**동기:** 캔버스가 기능적·디자인적으로 과해졌다. 사용자는 "필요한 노드를 클릭만 하면 추가되고, 중요도·확신만 심플하게 조정" 하기를 원한다.

## 핵심 사실 (근거)

- 최종 방향/판정(상승·하락)은 `forge-core.js`의 `aggregateConviction(graph)`가 **모든 노드의 확신×중요도를 종합**해 결정한다 (`forge-core.js:778-788`). **연결선과 무관.**
- 연결선(edges)은 예측의 "기준 시계열" 선택에만 쓰이고, 없으면 `combine` → 원본 가격으로 자동 폴백한다 (`forge-core.js:794-808`).
- 따라서 **연결선을 UI에서 제거해도 분석은 그대로 작동**한다. 안전을 위해 계산 시점에만 내부적으로 "전부 연결됨"으로 간주한다.
- 관련 메모: `scoopforge-edges-cosmetic`(엣지는 분석에 거의 무영향), `scoopforge-frontend-first`(웹 UI 품질 우선).

## 목표

1. 노드 추가 = 클릭 (현행 유지·미세 개선).
2. 중요도·확신 = **노드 카드에서 바로** 조정.
3. 연결선 개념을 사용자에게서 **완전히 숨김**.
4. 오른쪽 편집 패널은 **데이터 입력 전용**으로 대폭 축소.
5. **분석 결과는 변하지 않는다** (자동 완전연결로 엔진 입력 보존).

## 설계

### 1. 연결선 완전 제거 (UI·상호작용)

**제거 대상 (UI/상호작용만):**
- 노드 4면 포트 `.b-port` (마크업·CSS·hover 표시).
- 포트 드래그 연결 상호작용: `startLink`, `startEndpoint`, 포트 관련 포인터 디스패치.
- 엣지 렌더: `paintEdges`, `edgeGeo`, `#edgeG` SVG 그리기.
- 엣지 HUD: `drawEhud`, 엣지 선택(`selEdge`, `selectEdge`), 엣지 삭제 UI.
- 4점 자석/포트 스냅 중 **연결용** 경로(`portSnap`, `snapAt`, 링크 하이라이트).

**유지:**
- `boardState.edges` 데이터 필드는 유지(저장 포맷 하위호환). 단 화면엔 그리지 않고, 신규 사용자 상호작용으로는 생성되지 않는다.

**자동 완전연결 (핵심):**
- `boardToGraph()`가 계산 직전 **암묵적 연결**을 합성한다:
  - 모든 방향성 지표 블록(price/ma/rsi/trend/fib/elliott/phasefold 등, `volume`·`ticker` 제외) → `combine`(있으면) → `predict`(있으면).
  - `combine`/`predict`가 없으면 엣지 없이 반환(엔진이 원본 가격으로 폴백 — 기존 동작과 동일).
- 결과: 화면엔 선이 하나도 없지만 엔진은 "모든 지표가 예측에 반영됨"으로 계산 → **분석 결과 보존.**

**정리(cleanup):**
- `가중결합` 블록의 "연결선별 가중치 입력"(`combine` 파라미터 rows, `n.params.weights`) 제거 — 종합은 이미 확신×중요도로 처리됨.
- `autoLayout`에서 엣지 side 갱신 로직은 무해하므로 유지 가능(엣지 배열이 비면 no-op). 단 포트/엣지 렌더 호출 경로는 제거.

### 2. 중요도·확신을 노드 카드로

- 노드가 **단일 선택**되면 카드에 인라인 미니 컨트롤 표시:
  - `중요도`: 슬라이더 0–100 (기본 50).
  - `확신`: 슬라이더 −100 ~ +100 (기본 0).
- 슬라이더 조작 시 실시간으로 `n.weight` / `n.conviction` 갱신 → 디바운스 재분석(`fireBoardChange`) + 기존 중요도 배지(`b-n-wt`) 즉시 반영(`applyNodeWeightVisual` 재사용).
- 카드 위에서 편집 중엔 드래그/텍스트편집과 충돌하지 않도록 컨트롤 영역은 pointer 이벤트를 자체 처리(`stopPropagation`).
- 다중 선택 시엔 인라인 컨트롤 숨김(모호함 방지).

### 3. 오른쪽 편집 패널 대폭 축소

- **제거:** "가중치" 섹션(중요도/확신 → 카드로 이동), 가중결합 가중치 rows.
- **유지(간결):** 제목, 데이터 입력(가격 종가 붙여넣기 · 티커 심볼/현재가/주기 · 파라미터 1개 예: RSI 기간 · 이미지 보정), 이미지(선택), 서술 메모, 노드 삭제.
- 데이터/파라미터가 전혀 없는 노드(예: 추세선)는 패널이 거의 비게 되므로, 이 경우 패널 대신 "이 노드는 추가 설정이 없습니다" 정도의 최소 표시(또는 패널 미표시). 구현 판단은 계획 단계에서 확정.

### 4. 노드 추가 = 클릭 (현행 유지)

- 사이드바 "블록 도구" 클릭 → `addBlock(type)`이 캔버스 중앙에 추가(현행). 유지.
- 미세 개선: 새 노드가 기존 노드와 정확히 겹치지 않도록 위치를 소폭 오프셋(계단식).

## 영향 범위

- `map/forge.html`만 수정. `forge-core.js`는 **수정 없음**(입력 계약 유지).
- `forge-core.test.js` 영향 없음(엔진 미변경).
- 저장 포맷: `edges` 필드 유지 → 기존 서버/로컬 문서 하위호환. 기존 문서를 열면 저장돼 있던 엣지는 무시(안 그림), 재저장 시 비워질 수 있음.

## 비목표 (YAGNI)

- 노드 간 새로운 연결 UI(대체 시각화 등) 추가하지 않음.
- 엔진(forge-core) 로직 변경하지 않음.
- 자동 레이아웃/그룹 등 기존 기능의 재설계 없음(연결선 관련 호출 정리에 한함).

## 검증

- 샘플 포지(BTC/USD) 로드 → 판정/예측 콘이 이전과 **동일하게** 나오는지 확인(자동연결이 기존 수동연결과 같은 입력을 생성).
- 노드 추가 → 카드 인라인 슬라이더로 중요도·확신 조정 → 재분석 반영 확인.
- 편집 패널에 가중치 섹션이 사라지고 데이터 입력만 남았는지 확인.
- 화면에 포트·연결선·엣지 HUD가 전혀 나타나지 않는지 확인.
- 기존 저장 문서(엣지 포함) 로드 시 오류 없이 정상 표시.
