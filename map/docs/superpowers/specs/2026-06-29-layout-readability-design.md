# 결과 패널 확대 + 작도 가독성 + 시연 오버플로 — 설계

- 작성일: 2026-06-29
- 대상: 스쿱포지(Scoop Forge) — `forge.html` (레이아웃·작도 렌더링)
- 상태: 설계 승인됨 (구현 계획 대기)

## 1. 배경 / 문제

- 오른쪽 결과 패널(`.chart-pane`, 기본 50%)이 좁아 **차트 안 작도값(추세/MA/피보 라벨·선)이 빽빽**해 가독성이 떨어진다.
- 사이드바(`.forge-side`, 168px)는 이미지 라이브러리 썸네일 그리드를 상시 노출하고, 블록 도구 팔레트(`.forge-palette`)는 캔버스 위에 떠 있어 공간을 점유한다.
- **시연(재생) 중 작도값이 hero 경계 밖으로 나가 잘린다**(`.fc-hero{overflow:hidden}`). 특히 상단·우측 라벨과 재생 줌(predZoomIn) 시.

## 2. 목표

오른쪽 결과 패널을 키워 작도 가독성을 높이고, 사이드바를 도구 전용으로 정리하며, 시연 중 작도 잘림을 없앤다.

### 사용자 결정(확정)
- 사이드바: 이미지 라이브러리 **썸네일 그리드 제거**, `＋ 이미지` 추가 버튼만 유지. **블록 도구를 사이드바에 세로 배치**(떠있는 팔레트 제거).
- 캔버스(보드) 기본 폭 = **사이드바 폭 × 4 (≈640px)**. 결과 패널이 나머지 전부를 차지(=커짐).
- 예측 시그널을 **차트 패널 헤더에 인라인 통합**(별도 바 제거) → 분석 차트 위 여백 최소.

### 비목표 (YAGNI)
- 결과 패널 전체화면 토글(불요 — 분할 비율로 충분).
- 작도 알고리즘 변경(렌더 위치·가독성만; 분석 수학 불변).

## 3. 아키텍처

순수 CSS/마크업 레이아웃 변경 + 작도 라벨 좌표 클램프(렌더 단계). 분석 코어(`forge-core.js`)는 **불변**. 모든 변경은 `forge.html`.

## 4. 사이드바 재구성 (`.forge-side`)

- **이미지 라이브러리 그리드 제거**: `renderSide()`(또는 사이드바 렌더)에서 `lib-grid`(썸네일 목록) 마크업을 제거. `＋ 이미지` 추가 버튼(`libAddBtn`)과 그 핸들러는 유지. 드롭/Ctrl+V로 가격 노드에 이미지 추가하는 기존 동선 보존.
- **블록 도구 세로 배치**: 현재 `.forge-palette`(캔버스 위 `position:absolute`)의 버튼들(기본/중간 노드·아이콘 등)을 사이드바 섹션(`.side-sec`)으로 이동, 세로 리스트(`.pal-btn` 재사용 또는 사이드바 스타일). 떠있는 `.forge-palette`는 제거(또는 `display:none`).
- 사이드바 폭 ~160px 유지(세로 버튼 가독). `--side-w: 160px` 토큰화 권장(섹션 5 배수 계산에 사용).

## 5. 레이아웃 분할 (보드 축소 · 결과 패널 확대)

- `.board-pane`: `flex:1 1 auto` → **`flex:0 0 var(--board-w, 640px)`** (기본 640 ≈ 사이드바 160×4). `min-width:280px` 유지.
- `.chart-pane`: `flex:0 0 50%` → **`flex:1 1 auto`** (나머지 전부 차지). `min-width:0` 유지.
- **구분선 드래그**: `forge-gutter` 핸들러가 기존엔 chart-pane 폭(%)을 조절했다면, 이제 **`--board-w`(px)를 조절**하도록 변경. 더블클릭 초기화 = 640px.
- **영속화**: `--board-w`를 localStorage(`scoopforge_board_w`)에 저장, 부팅 시 복원. (storeMode 무관 — 순수 UI 환경설정. try/catch로 안전.)

## 6. 예측 시그널 인라인 통합

- `verdictBadge`(별도 `.verdict-badge` 바, 마크업 line ~501)와 그 활성 표시 제거.
- `renderVerdict(verdict)`가 채우던 내용(국면·시그널·목표가)을 **hero 패널 헤더(`fc-phead`, "가격 차트" 줄)의 우측 영역(`fc-head-actions` 앞)에 컴팩트 readout**으로 렌더. 예: `국면 ▲상승 · 시그널 +42 · 목표 71,234`. 색은 국면별(bull/bear/eth) 토큰.
- 편집 모드(`chart-pane.editing`)에선 기존처럼 숨김 처리 일관 유지.
- 결과: 차트 위 별도 바가 사라져 차트가 헤더 바로 아래 붙음(여백 최소).

## 7. 작도 가독성 + 시연 오버플로

### 7.1 hero 기본 높이
- `--hero-h` 기본 440 → **560px**. fcVGutter 드래그·영속 유지(기존 로직 그대로).

### 7.2 라벨 가독성
- 작도 라벨(추세 "추세"/MA 배열·크로스/피보 ratio·목표가/범례 등) 렌더 시, 텍스트 뒤에 **반투명 다크 배경 pill**(`rgba(11,15,20,.66)` 라운드 박스)을 깔아 캔들·그리드 위 대비 확보. 공용 헬퍼 `_evLabel(c, text, x, y, color, align)` 신설 — 배경 박스 + 텍스트를 함께 그림. 기존 `c.fillText` 라벨 호출을 이 헬퍼로 교체(추세/MA/피보 작도 + 범례).
- 폰트 9px → **10px**(라벨), 범례는 현행 유지 또는 +1px.

### 7.3 오버플로 클램프 (시연 핵심)
- 라벨 좌표를 캔버스 경계 안으로 클램프하는 공용 로직을 `_evLabel`에 내장:
  - x: `[M, W - textWidth - M]` 범위로 클램프(M=마진 3px). 우정렬 라벨은 우경계 안쪽.
  - y: `[M + fontAsc, H - M]` 범위로 클램프. 상단 근접(예: 선이 위 경계에 붙어 라벨이 `y-2`로 음수) 시 **선 아래로** 내려 그림.
- 선/마커도 좌표가 캔버스 밖이면 그리지 않거나 경계로 클램프(기존 `isFinite`·`clipY` 보강): 라벨이 핵심이므로 라벨 클램프를 우선.
- 재생 줌(`predZoomIn`)이 hero에 transform scale을 걸 때 작도 캔버스도 함께 스케일되어 라벨이 경계 밖으로 밀리는 경우 → 라벨 클램프가 화면 좌표 기준으로 동작하므로 자연 완화. (추가로 재생 중에는 라벨 클램프 마진을 약간 키워 안전.)

## 8. 검증

- 레이아웃·캔버스라 단위테스트 영역은 적음. **구문검사(`node --check` on inline script)** + 헤드리스/라이브 시각검증(컨트롤러).
- 클램프 헬퍼는 순수 좌표 함수로 분리 시 단위검증 가능: 입력 (text, x, y, W, H) → 반환 좌표가 `[M, W-M]·[M, H-M]` 내. (forge.html 내부 함수라 node 테스트 어려우면 시각검증으로 갈음.)
- 시각검증 항목: ① 사이드바 세로 도구·라이브러리 그리드 없음 ② 보드≈640·결과 패널 큼·구분선 영속 ③ 차트 헤더에 시그널 인라인·별도 바 없음 ④ hero 560·라벨 pill 또렷 ⑤ ▷ 엔진분석 재생 중 작도 라벨이 경계 안에 머묾(잘림 없음).

## 9. 영향 / 호환

- `forge-core.js`·데이터 모델·`map.html` 불변. 변경은 `forge.html`(CSS·마크업·렌더 함수).
- 기존 리사이즈(forge-gutter, fcVGutter)·영속 패턴 재사용.
- 단일 HTML·바닐라 JS·무빌드·다크 토큰·한국어·noindex 유지.
- 떠있는 팔레트 제거 시 관련 핸들러(`renderToolbar`/팔레트 버튼) 사이드바로 이전 — 기능 동등 유지.
