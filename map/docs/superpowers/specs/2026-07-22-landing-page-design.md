# 랜딩 페이지 설계 (map/index.html)

- 날짜: 2026-07-22
- 대상: `map/index.html` (신규)
- 출처 시안: `Forge.html 디자인 개선안.zip` → `Naru Landing.dc.html`

## 1. 배경

시안 zip은 forge를 "나루(Naru) by 머니스쿱"으로 리브랜드한 전제의 디자인 문서 4건(Landing·Basic·Auth·Redesign 3방향)이다. 이 스펙은 그중 **랜딩 1건만** 다룬다. 나머지 3건은 별도 사이클.

### 시안을 그대로 쓸 수 없는 이유

시안 히어로의 주장은 "33개 지표가 내리는 하나의 결론 · 방향 적중률 62% · 상승 확률 68%" — **방향 예측 도구**의 서사다.
그러나 `forge-scorecard.html` 실측은 정반대다:

- 방향 적중 **58.1%** vs 기준선(항상 상승) **60.8%** → **미달**
- "이 엔진은 방향 예측 도구가 아니라 국면 파악·진입 타이밍·리스크 가늠 도구다"

"우리는 성적을 숨기지 않습니다" 섹션을 단 페이지가 히어로에서 미검증 방향 적중률을 내세우면, 스코어카드 한 번만 눌러도 자기모순이 드러난다. 따라서 **레이아웃은 채용하되 서사는 교체**한다.

## 2. 결정 사항

| 항목 | 결정 |
|---|---|
| 파일 | `map/index.html` — `/map/` 의 정문 |
| 브랜드명 | **미정**. `BRAND` 상수 1곳으로 격리해 한 줄 교체 가능하게 |
| 테마 | 라이트(기본) + 다크 양쪽. 토글 + `prefers-color-scheme` + localStorage |
| 액센트 | 시안 블루(`#1f6feb` / 다크 `#3b82f6`). 골드 회귀는 토큰 한 줄 |
| 서사 | **A안 — 정직한 역제**: "우리는 방향을 맞힌다고 말하지 않습니다" |
| 카피 사실성 | 현재 사실만. 가격·플랜 제한·미검증 수치 전량 삭제 |
| 검색 노출 | `noindex, nofollow` 유지 (유사투자자문업 신고 전 베타) |

## 3. 기술 제약 (프로젝트 관례)

- 빌드 도구 없음. 순수 HTML·CSS·바닐라 JS. 단일 자립 파일.
- 외부 의존성은 Pretendard CDN 하나만 (`map.html` 선례).
- CSS는 `<style>` 인라인, `forge-pricing.html` 패턴.
- **좌측 컬러 accent bar/rail 금지** (프로젝트 전역 금지 규칙).
- 하드코딩 색상 금지 — `:root` 토큰만.

## 4. 디자인 토큰

시안 값을 그대로 채용한다.

```
:root        --bg:#f4f2ec  --bg-hi:#faf6ec  --surface:#fbfaf6  --surface-2:#efece3
             --tint:#efe9dd  --footer-bg:#ece4d3  --border:#e4dfd3  --border-2:#d8d1c1
             --text:#2a2620  --muted:#5a5449  --muted-2:#7a7364  --muted-3:#9a9284
             --accent:#1f6feb  --accent-strong:#1a5cc4  --on-accent:#fff
[data-theme=dark]
             --bg:#0f0d0a  --bg-hi:#151109  --surface:#16130e  --surface-2:#1c1913
             --tint:#100e0a  --footer-bg:#0c0a07  --border:#26221b  --border-2:#332e24
             --text:#f4f2ec  --muted:#b8b0a0  --muted-2:#a49c8c  --muted-3:#8f8878
             --accent:#3b82f6  --accent-strong:#60a5fa  --on-accent:#fff
```

**양 테마가 같은 키를 전부 지정한다** (forge `THEMES` 정책과 동일 — 누락 시 전환 간 leak).

테마 초기화 순서: localStorage(`site_theme`) → 없으면 `prefers-color-scheme` → 없으면 light.
FOUC 방지를 위해 테마 판정 스크립트는 `<head>` 인라인으로 두어 첫 페인트 전에 `data-theme`을 확정한다.

## 5. 섹션 구성

### 5.1 헤더 (sticky)
좌: 브랜드 워드마크. 중: 작동 방식 · 검증 성적 · 요금제 (앵커/링크). 우: 테마 토글 + `분석 시작`.
`로그인` 버튼은 **미노출** — 인증(`forge-auth.php`)이 아직 비활성.

### 5.2 지원 시장 스트립
시안은 `NVDA 172.40 +2.1%` 형태의 더미 시세를 마퀴로 흘린다. **가격 숫자를 제거**하고 종목명·티커·시장군만 흐르게 한다.
근거: 정직을 내세운 페이지 상단의 가짜 시세는 자충수. 실시간 연동은 `forge-api.php` 의존이 생겨 v1 범위 밖.
`prefers-reduced-motion: reduce` 시 애니메이션 정지.

### 5.3 히어로
```
우리는 방향을 맞힌다고 말하지 않습니다.

상승·하락 예측은 시장 기준선을 넘지 못합니다 — 저희도, 다른 어느 기술적 분석도.
대신 얼마나 움직일지는 맞힙니다.

[ 무료로 분석 시작 → ]  [ 검증 성적 보기 ]

  69.0%          67~69%         1.9%p
  변동성 예보     낙폭 위험       확률 오차

20년 · 54종 · 93k시점 walk-forward 검증
```

### 5.4 제품 모형
시안의 종목 판정 카드를 유지하되 **"예시 화면" 라벨을 명시**한다. 실제 시세로 오인될 여지를 없앤다.
표시 항목은 검증된 축으로 교체: 국면 배지 · 변동성 예보 · 낙폭 위험 · 진입/손절/목표 · 확률 R:R.

### 5.5 맞히는 것 / 못 맞히는 것
✓/✕ 2열 대비. 랜딩의 핵심 차별점이자 A안 서사의 증명 구간.

| ✓ 검증됨 | 수치 |
|---|---|
| 변동성 예보 (2주·1달·2달) | 69.0% (기준선 대비 +17.5%p) |
| 낙폭 위험곡선 (1·2·3개월) | 67~69% |
| 이익목표 도달 | 63~65% |
| 급변 경보 (2σ·2.5σ·3σ) | 65% |
| 갭 경보 (주식) | 63% |
| 확률 캘리브레이션 (OOS ECE) | 1.9%p |
| 지지반등 진입 신호 | 승률 53~56% |

| ✕ 못 하는 것 |
|---|
| 방향 예측 — 58.1%, 기준선 60.8% 미달 |
| 인터마켓 · 내부자거래 · 공매도잔고 · PEAD — 검증 후 기각 |

### 5.6 작동 방식 3단계
01 종목 검색 → 02 판정 확인 → 03 결과로 검증. 시안 카피 유지하되 02의 "상승 확률" 표현을 "국면·변동성·리스크"로 교체.

### 5.7 검증 방법론
walk-forward(미래 차단) · 표본·기간·기준선 병기 원칙 · 기각 이력 공개. `forge-scorecard.html` 링크.

### 5.8 요금
"현재 전체 베타 — 무료" 한 줄. 가격 숫자·카드·플랜 제한 언급 없음. `forge-pricing.html` 링크.

### 5.9 최종 CTA + 푸터
푸터에는 **실재하는 링크만**: 작동 방식(`forge-guide.html`) · 검증 성적(`forge-scorecard.html`) · 요금제(`forge-pricing.html`).
블로그 · 문의 · 이용약관 · 개인정보처리방침은 페이지가 없으므로 **제외**.
투자 유의 안내 전문을 시안 그대로 유지 (유사투자자문업 관련 필수 고지).

## 7. 반응형

기존 forge 관례에 맞춰 모바일 대응. 860px 이하에서 히어로 세로 스택 · KPI 3열→1열 · 헤더 내비 축약 · ✓/✕ 2열→1열.

## 8. 접근성

- 테마 양쪽에서 본문 대비 4.5:1 이상.
- 마퀴는 `prefers-reduced-motion` 존중.
- CTA는 `<a>`, 토글은 `<button aria-pressed>`.
- 히어로 KPI 수치는 스크린리더용 단위 텍스트 동반.

## 9. 하지 않는 것 (YAGNI)

로그인 폼 · 결제 · 실시간 시세 연동 · 블로그/약관 페이지 · forge 본체 리디자인 · Basic/Auth/Redesign 시안 적용.

## 10. 검증

- 헤드리스 스크린샷(라이트·다크 × 데스크톱·모바일 4장)으로 육안 확인. [[headless-verify-wsl]]
- 페이지 내 모든 수치가 `forge-scorecard.html` 실측치와 일치하는지 대조.
- 죽은 링크 0건 확인.
- `noindex` 메타 존재 확인.
