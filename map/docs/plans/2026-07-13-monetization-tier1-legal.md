# 스쿱포지 유료화 티어1 — 법적 검토 브리핑 (유사투자자문업)

작성 2026-07-13 · 웹 조사 기반 브리핑. **법률 자문이 아님 — 유료화 개시 전 자본시장 전문 변호사/행정사 확인 필수.**

## 1. 규제 지형 (자본시장법 §101 + 2024~25 개정)

- **유사투자자문업 정의**: 투자자문업자 외의 자가 **대가를 받고** 간행물·통신물·방송 등으로 **불특정 다수**에게 금융투자상품의 **투자판단·가치에 관한 조언**을 업으로 하는 것 → 금융위(금감원) **신고제**.
- **2024~25 개정(리딩방 규제) 핵심**:
  - 허용 범위 = **일방향 채널·비개별 조언**만. 오픈채팅·SNS 등 **양방향 유료 채널 운영은 투자자문업(등록제)으로 규율** — 유사투자자문업 신고로는 불가.
  - 진입 강화: 대표자 금융투자협회 교육 이수 필수, 금융관련법 + 방판법·전자상거래법 위반 이력 시 진입 제한. 자본금 요건은 없음.
  - **손실보전·수익보장 약속 금지** — 위반 시 3년 이하 징역/1억 이하 벌금.
  - 퇴출 강화: 시정조치 불이행·5년 내 반복 제재 시 직권말소.
- **단순 정보 제공과의 경계(금융위 유권해석)**: 금융 지식·데이터 제공 수준이면 신고 불요. 그러나 **특정 종목의 투자판단·가치에 관한 조언을 대가 받고 제공하면 해당**. **무료면 "대가 수취" 요건 미충족 → 신고 대상 아님.**
- **소프트웨어 판례(인천지법 2017노1649)**: 자동화 시스템(프로그램) 자체는 "인간 간 의사소통"이 아니어서 **투자자문업(등록제·형사처벌)에는 불포섭**. 다만 불특정 다수에 전략·신호를 제공하는 형태는 **유사투자자문업 신고 대상**.

## 2. 스쿱포지 해당성 분석 (정직 평가)

| 요소 | 현황 | 판정 소지 |
|---|---|---|
| 특정 종목 상승확률·목표가·"지지반등 매수 후보" 배지 | 있음 | 단순 정보를 넘어 **투자판단 조언**으로 해석될 소지 높음 |
| 대가 수취 | 현재 무료(베타) | **현재는 신고 불요** — 유료화 시점 = 신고 필요 시점 |
| 채널 형태 | 셀프서브 웹 도구·일방향·비개별(채팅/1:1 없음) | 개정법 허용 형태에 부합 ✓ |
| 수익보장·손실보전 표현 | 없음 — 오히려 전면 면책·약점 공개(스코어카드) | 준수 ✓ (기존 정직 노선이 규제 자산) |
| "엔진분석(클로드 수동분석·예정)" 기능 | 미구현 | **1:1 개별 분석으로 설계하면 투자자문업(등록제) 영역 침범 위험** — 비개별 자동 산출로 한정 설계 필요 |

## 3. 경로 선택지

- **A안(권장): 유사투자자문업 신고 후 유료화.** 절차 = 대표자 협회 교육 이수 → 금감원 신고(자본금 요건 없음·개인/법인 가능) → 신고번호 광고 표기 등 영업규제 준수. 현 제품 표현(확률·목표가·기회 배지)을 그대로 유지 가능. 유의: 양방향 유료 채널 금지 유지, 환불(전자상거래법·통신판매업 신고) 병행.
- **B안: 신고 없이 "데이터·도구" 포지셔닝 유지.** 매수 후보·목표가·확률 등 조언성 표현을 전면 중립화해야 하고 경계가 회색 — 제품 가치 훼손 크고 리스크 잔존. **비권장.**
- 공통: 현 무료 베타는 신고 없이 적법 여지(대가 없음) — 유료 전환 전에만 A안 완료하면 됨. **유료/무료 구분을 맨 마지막에 정한다는 운영 지침과 정합**(기능은 계속 만들고, 신고를 유료화 게이트로).

## 4. 전문가 확인 필수 항목

1. 스쿱포지 산출물(확률·목표가·기회 배지)의 "조언" 해당성 최종 판단 — B안 여지 유무.
2. 신고 주체(개인 vs 법인) 및 겸업 이슈, 통신판매업 신고와의 병행.
3. 자동 알림(브라우저 푸시·향후 이메일)의 "일방향·비개별" 요건 충족 형태.
4. "엔진분석(수동분석)" 기능의 설계 한계선(비개별 유지 조건).
5. 광고·표시 의무(신고번호, 위험고지 문구 표준), 라이브 트랙레코드/스코어카드 표기의 광고 규제 저촉 여부.
6. 해외 사용자 대상 판매 시 관할.

## 5. 다음 단계 (티어1 나머지)

- **인증**: Google OAuth(헤더 버튼 자리 이미 있음) — cafe24 PHP 서버 세션 + `forge_users.json`(또는 SQLite). 문서/워치리스트의 사용자별 분리(현재 전역 공유 → 로그인 도입 시 마이그레이션 설계 필요).
- **결제**: 국내 구독 = 토스페이먼츠/포트원 정기결제 API(PHP 웹훅 수신 가능). 요건: 통신판매업 신고 + (유료화 시) 유사투자자문업 신고 선행.
- 순서 제안: 인증(로그인·사용자별 데이터) 먼저 → 신고 절차 병행 → 결제 붙이기 → 마지막에 유료/무료 구분(지침).
