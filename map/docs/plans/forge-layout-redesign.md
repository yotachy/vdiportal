# Scoop Forge 레이아웃 개편 (지표 선택 통합 + 종목 워치리스트)

> 세션 끊겨도 이어서 완료. 사용자 확정 사항 기준. 대상: `map/forge.html`. 배포: git+cafe24 www/map/.

## 배경
사용자: "블록도구(좌측)와 범례(차트) 두 곳 선택이 직관적이지 않고 불편. 하나로 통합하되 차트 옆으로. 포지→종목명, 미니판정 표기."

## 확정된 방향
- **1단계(완료·배포 4a32b1f)**: 포지→**종목** rename, 항목명=티커(_docTicker)+미니판정 배지(d._verdict, ▲97%), 사이드바 168→200px, 모바일 칩·콤보도 티커.
- **2단계(완료·배포)**: 지표 레일 = board·chart 사이 세로 체크박스(지표 조합). 1차=전체 지표 항상(_ensureAllInd 19종), 레일 체크=표시·2차(_evVisible), 더블클릭=단독. 좌측 블록도구·데스크톱 차트범례 제거(모바일 범례 유지). — 원문↓
- **2단계(구)**: **지표 선택 통합 레일 = 가운데(board-pane)·차트(chart-pane) 사이 세로 레일**. 좌측 블록도구 + 차트 플로팅 범례를 이 레일 하나로 대체.
- **3단계(완료)**: 종목=워치리스트(티커명+미니판정+드래그정렬), ＋종목추가 시 티커입력 자동포커스.
- **4단계(완료)**: 분석 프리셋 — 기본4(전체종합·추세중심·모멘텀중심·스마트머니)+사용자저장(localStorage scoopforge_presets). _applyPreset→_evVisible. 레일: 전체포함 버튼 상단, 편집버튼 토글(재클릭=분석복귀), 노드삭제 제거(전체지표 항상).

## 2단계 모델 (핵심 결정 — 구현 시 확인)
**추천: 1차=전체 고정 / 레일=2차·표시 부분집합**
- **1차(종합)** = 항상 전체 지표(~19) 융합. 고정.
- **레일 체크박스** = 차트 표시 + 2차(선택조합) 포함. 체크 해제한 지표는 1차엔 남고 2차/표시에서 빠짐.
- → 1차≠2차 구분 성립. 선택 한 곳. 차트 바로 옆.
- 구현: 1차 run이 항상 전체 지표 그래프 사용해야 함(현재는 boardToGraph=추가된 것만). 로드 시 전체 지표 auto-add 또는 1차용 풀그래프 구성.
- `_evVisible`가 곧 레일 체크 상태(표시+2차). 더블클릭=단독(_focusInd) 유지 가능.

대안: 레일 체크=1차 활성(add/remove) + 2차는 행별 2차토글(2상태). → 복잡, 비추천.

## 2단계 구현 단계
1. `.forge-split` 안에 `board-pane`과 `chart-pane` 사이 `.ind-rail` DOM 추가(세로 체크박스 목록). `renderIndRail()`.
2. 블록도구 렌더(pal-side)를 레일로 이관, 체크=_evVisible 토글(표시+2차). 좌측 블록도구 섹션 제거.
3. 차트 플로팅 범례(_evLegend) 제거 또는 색 키로 축소(레일이 대체).
4. 1차=전체: 전체 지표 auto-add(또는 풀그래프). 2차=_evVisible.
5. 모바일: 레일을 접이식/드로어로(현 mSub 흐름과 정합).
6. 검증: 체크 토글→2차 갱신·표시 반영, 1차 불변. 129 테스트 유지.
