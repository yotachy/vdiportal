# PotFlow — 책갈피 노드 UX·자동화·썸네일 수정 · 설계

- 작성일: 2026-07-11
- 대상: `map/potflow.html`

---

## A. 썸네일 액박(깨진 이미지) 수정 [버그]
원인: `imgSrc(id)`가 이미지 없으면 `''` 반환 → `<img src="">`가 액박. + 로컬 저장(`userImagesOnly`)이 `bm_*`(책갈피 썸네일) 미저장 → 새로고침 시 유실.
- `nodeHTML`: 썸네일 `<img>`는 **`n.thumb && imgSrc(n.thumb.imgId)`가 유효할 때만** 렌더, 아니면 드롭 플레이스홀더. (src 빈값 방지)
- `userImagesOnly`: `vthumb_`뿐 아니라 **`bm_`도 저장**. (책갈피 썸네일 영속)

## B. 책갈피 노드 하위 스타일 + 시간줄 제거 + 썸네일 확대
책갈피 노드(=`bmParent!=null`)를 원본과 시각적으로 차별(하위 느낌):
- `nodeHTML`에 **책갈피 분기**: `🔖` 배지(편집 title 밖) + 골드-딤 테두리 + 약간 작은 폭(하위 느낌). **desc(시간) 줄 제거**(공간 안 씀). **썸네일 높이 확대**(`.bm-thumb` 예: 120px+).
- 좌측 accent 라인 금지 — 배지·테두리·배경으로만 구분.

## C. 책갈피 자동 추가(수동 버튼 불필요) + 자동 정렬
- **자동 트리거**: 영상 지정 시(기존 `bindVideoToNode`)·PotPlayer 닫힐 때(기존 `watchClose`) + **부팅/캔버스 로드 시 헬퍼 온라인이면 루트 영상 노드 전부 `syncBookmarks` 1회**(재방문·인코딩 수정 반영 자동). 수동 "책갈피" 버튼은 유지(강제 새로고침용).
- **자동 정렬**: `syncBookmarks`가 체인 배치 시 **부모 기준으로 항상 깔끔히 재배치**(부모 아래/오른쪽 균등 간격, 겹침 없음). 신규뿐 아니라 재동기화 때도 정렬(사용자 위치 유지보다 정렬 우선 — 하위 노드 한정).

## D. 그룹 이동 시 멤버 노드 동반 이동
- 그룹 박스(`.group`) 본체 드래그 → **그룹 소속 노드 전부 함께 이동**. pointerdown이 그룹 박스(버튼·라벨 제외)에서 시작하면 그룹 드래그 모드: onMove에서 dx/dy를 멤버 노드 x/y에 적용. 드롭 시 markDirty.

## E. 노드 클릭 → 우측 파일 목록에서 해당 제목 깜빡임
- 노드 단일 선택 시 그 노드의 `videoPath`와 일치하는 오른쪽 목록 행(`.rs-file[data-path]`)에 **`.rs-blink` 클래스**(CSS 애니메이션 1~2회)로 강조 + `scrollIntoView`. 현재 열린 폴더에 그 파일이 있을 때만.

## 디자인/제약
