# 스쿱랩 (Scoop Lab) — 설계 문서

- 날짜: 2026-06-26
- 브랜드: MoneyScoop 패밀리 (Scoop Board · PHASE-FOLD · ScoopSignal과 동일 팔레트)
- 상태: 컨셉 합의 완료 → MVP 설계
- 워킹 네임: **스쿱랩 / Scoop Lab** (확정 아님, 검토 시 변경 가능)

---

## 1. 한 줄 정의

**당신의 투자 사고(철학+기술+주기)를 노드로 "작곡"하고, 실행하면 시장 데이터 위에서 살아 움직이며 미래를 가리키는 분석 악기.** 스쿱보드(악보)와 PHASE-FOLD(연주)를 한 화면에서 잇는 머니스쿱 플래그십.

## 2. 정체성 / "괴랄+감탄"의 출처

- 차티스트가 보던 "남이 만든 지표"가 아니라, **내가 그린 사고 구조 그 자체가 화면에서 신호를 낸다.**
- 시그니처 장면(세 가지가 **한 몸**으로 동시 발생):
  1. **폴딩** — 각 블록의 파동이 PHASE-FOLD(epoch folding)로 *접혀* 주기 위에 겹친다. "지표를 그리지 마라, 파동을 접어라."
  2. **살아있는 그래프** — 보드에서 그린 연결선을 따라 신호가 맥동(pulse)하며 흘러 차트 안까지 뻗는다. 배선이 곧 차트 위 빛줄기.
  3. **합의 필드** — 흘러온 신호들이 차트 위에서 간섭무늬처럼 수렴/발산. 꽉 겹치면 *확신 밴드*가 빛나고 어긋나면 *경고*. 합의가 앞으로 **예측 콘(cone)**으로 돌출되고, 분기 노드에서 시나리오 A/B로 갈라지며 각각 목표가·무효화선을 긋는다.
- 한 줄 요약: **"내 추론의 겹침"을 물리적 파동 간섭으로 보여주는 차트.** 아무도 이렇게 안 보여줬다(낯섦=괴랄), 그런데 말이 된다(감탄).

## 3. 범위 (Scope)

- **기존 두 페이지(`map/map.html`, `map/chart.html`)는 손대지 않는다.** 새 단일 HTML 페이지로 시작.
- 새 산출물: `map/lab.html` (단일 HTML, 빌드 도구 없음, 바닐라 JS — 프로젝트 관례 유지).
- v1 데이터: **데모 합성 파동 + 사용자 OHLC 임포트.** 실데이터 연동은 다음 단계 유료 훅(ScoopSignal `api.php` 프록시 재활용).
- 기본적 분석: 데이터 소싱 없음 — **사용자가 거는 "확신 가중치/바이어스"**로 합성 시그널에 반영(자유 노드 + 가중).

## 4. 아키텍처

새 페이지 = 통합 셸. 세 레이어.

```
┌─ lab.html (split view) ─────────────────────────────────┐
│  [보드 페인 (좌)]      ▷실행      [라이브 차트 페인 (우)] │
│   노드/엣지 에디터   ──────────►   폴딩·합의·예측 렌더     │
│        │                                    ▲             │
│        └──────► [인터프리터 코어] ──────────┘             │
│                 strategy graph → 계산 DAG → 결과          │
└──────────────────────────────────────────────────────────┘
                       ▲
                [데이터 레이어]  데모 생성기 / OHLC 임포트
```

### 4.1 보드 페인 (좌) — Scoop Board 엔진 재사용·확장
- `map.html`의 캔버스/노드/엣지/팬·줌/HUD 렌더 로직을 가져와 **경량화**(빌트인 썸네일 base64 3.2MB는 **가져오지 않음** — lab은 이미지 노드 비중 낮음).
- 노드 모델 확장: 기존 자유 노드(`type: full/mini/icon`)에 더해 **실행 블록 노드**(`block` 명세 포함)를 추가. 자유 노드 = 맥락(계산 제외), 실행 블록 = 계산 대상.

### 4.2 인터프리터 코어 (신규 — 제품의 심장 / 핵심 IP)
- 입력: strategy graph(노드 + 엣지, 블록 명세 포함).
- 처리: 엣지를 따라 **계산 DAG** 구성 → 위상 정렬 → 활성 데이터셋에 대해 평가.
- 출력 계약(아래 §6).
- **순수 함수 집합으로 작성** — DOM 비의존, 결정적(deterministic). 테스트 가능성의 핵심.

### 4.3 차트 페인 (우) — PHASE-FOLD 렌더 재사용·확장
- `chart.html`의 폴딩 트레이스 / PDM 스펙트럼 / 예측 구간 렌더를 가져옴.
- 신규 오버레이: **살아있는 그래프(맥동)** + **합의 필드(간섭무늬)** + **시나리오 분기/콘**.

### 4.4 전략 계약 (Strategy JSON)
- Scoop Board의 기존 노드/엣지 모델의 **상위 집합(superset)**.
- 노드: `{ id, x, y, kind:"free"|"block", blockType?, params?, ... }`, 엣지: 기존 + 신호 흐름 의미.
- 이 JSON이 곧 (장차) 보드 "전략 내보내기" ↔ lab "불러오기" 교환 포맷이자 lab 내부 상태.

## 5. 노드 어휘 (하이브리드)

**실행 블록 패밀리** (계산 대상):
- **데이터 소스** — 가격/캔들, 오실레이터(데모·임포트). *MVP: 가격 1종.*
- **기술 변환** — 이동평균·RSI류 + **PHASE-FOLD 주기 블록**(시그니처). *MVP: MA + PHASE-FOLD.*
- **결합/가중** — 여러 신호를 가중 합성 → 합성 시그널. *MVP 포함.*
- **조건/분기** — "주기 정합도 > X" → 시나리오 A/B. *Phase 2.*
- **출력** — 예측 경로 / 시그널 스코어 / 시나리오 판정(목표가·무효화). *MVP: 예측 + 시그널.*

**자유 노드** (계산 제외, 맥락):
- 철학·근거 주석. **기본분석 노드**는 예외적으로 "확신 가중치"를 합성에 바이어스로 건다. *기본분석 바이어스는 Phase 2.*

## 6. 출력 계약 (실행 결과)

실행 시 인터프리터가 반환:
- `predictionPath` — 미래 N봉 예측 경로 1줄기 + 신뢰구간 음영(추세 + PHASE-FOLD 외삽).
- `signalSeries` — 합성 시그널 스코어 시계열(예: −100~+100), 임계 돌파 시 매수/매도 존.
- `verdict` — 현재 국면/시나리오 판정 { scenario, targetPrice, invalidation } *(Phase 2: 분기 기반 완성형, MVP는 단일 국면 요약)*.

## 7. MVP 분해 (한방에 "와우" 판가름)

목표: 분할 뷰에서 **작지만 진짜로 동작하는** 인터프리터가 데모 데이터 위에서 시그니처 장면을 그린다 → 컨셉의 WOW를 빠르게 검증.

- **MVP-1 (코어 증명)**
  - 분할 뷰 셸(보드 좌 / 차트 우 / ▷실행).
  - 블록 최소셋: 데이터소스(가격) · 기술(MA + PHASE-FOLD) · 결합(가중) · 출력(예측 + 시그널).
  - 인터프리터: 위 체인을 데모 데이터로 평가 → `predictionPath` + `signalSeries`.
  - 시그니처 장면: **폴딩 + 합의 밴드 + 예측 콘**을 실시간 렌더. 살아있는 그래프 맥동은 **기본 펄스 수준**(정교화는 후속).
- **MVP-1.5** — OHLC 임포트(붙여넣기/업로드) → 데모 대신 실제 데이터 위에서 동작.
- **Phase 2** — 조건/분기 → 시나리오 A/B + 목표가·무효화, 기본분석 확신 가중치, 살아있는 그래프 풀 연출.
- **Phase 3** — 실데이터 연동(유료 훅), 서버 저장(`api.php` 재활용), 전략 마켓/공유.

## 8. 리스크 / 주의

- **실시간 재계산 성능** — 편집마다 재계산은 무거움. 디바운스 + 인터프리터 경량 유지(웹워커 없음, 바닐라). 노드 수 상한 가드.
- **파일 크기 / 코드 병합** — 두 거대 HTML의 전역(global) 충돌. lab은 처음부터 **네임스페이스 정리**하며 필요한 로직만 발췌(특히 map의 3.2MB 내장 이미지 미포함).
- **cafe24 제약** — POST <128KB, 정적 파일 내 `<?` 리터럴 금지(WAF), localStorage는 로컬모드 opt-in — 기존 메모리/관례 준수.
- **noindex** — 새 페이지도 `<meta name="robots" content="noindex,nofollow">` 필수.

## 9. 테스트 전략

- 바닐라·무빌드 → **인터프리터 코어를 순수 함수로** 떼어 결정적 데모 데이터로 검증(인라인 어서션 하니스 또는 헤드리스).
- 시각 검증: WSL 헤드리스 playwright chromium 스크린샷(기존 메모리 절차) — 분할 뷰·폴딩·콘 렌더 확인.
