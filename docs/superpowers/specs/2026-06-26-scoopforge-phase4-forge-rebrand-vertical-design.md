# 스쿱포지 Phase 4-A — 포지 리브랜드 + 세로 정렬 설계

- 날짜: 2026-06-26
- 선행: Phase 3 A+B(서버 저장 + 사이드바) 배포 완료.
- 상태: 컨셉 합의 완료 → 구현 설계. **데이터 무관 빠른 사이클(R1+R3).**

## 0. 상위 맥락(로드맵)

사용자 대전환 요청을 분해 — 순서:
- **이번(R1+R3, 본 문서)**: 포지 리브랜드 + 세로 정렬(데이터 무관, 빠름).
- **다음(R2+R4+R5, 별도 사이클)**: 대표 이미지=우측 가격차트 + 파동스캔/엘리어트 등 전문 블록 + "포지 분석" 재생. **Claude 비전 베스트-에포트**(이미지가 품은 정보에 맞춰 데이터 있으면 정밀, 없으면 파동·추세 등 읽어 반영) — D(비전) 앞당김. 큰 작업.

(본 문서는 R1+R3만. R2/R4/R5/엘리어트/비전 결정은 §6에 기록.)

## 1. R1 — 포지 리브랜드 (UI 라벨만)

내부 식별자·서버 키·데이터 구조(`DOCS`/`documents`/`activeId`/`themeImgId`)는 **그대로 유지**(서버 호환). UI 텍스트만 변경:
- 사이드바 헤더 `전략 문서` → `포지` (forge.html:558).
- 새 포지 버튼 `＋ 새 문서` → `＋ New Forge` (forge.html:558).
- 신규 포지 기본 제목 `새 전략` → `New Forge` (boot 시드 forge.html:539, newDoc forge.html:565).
- 삭제 가드 토스트 `최소 1개 문서는 필요해요` → `최소 1개 포지는 필요해요` (forge.html:581).
- 내보내기 토스트 `전략 내보내기 완료` → `포지 내보내기 완료` (forge.html:1723).
- 시드 메모 노드 `전략 메모` → `포지 메모` (forge.html:1262, 일관성).
- (캔버스 단위 = "포지" 용어 통일. 그 외 사용자 가시 "전략/문서" 표현 발견 시 포지로.)

## 2. R3 — 세로 정렬 (map.html autoLayout 포팅)

forge에 autoLayout 없음(`fitView`만 존재). 신규 추가:
- `parentsOf(id)` 헬퍼: `boardState.edges`에서 `e.to===id`인 `e.from` 목록.
- `autoLayout(dir)`: map.html 알고리즘 포팅 — 노드를 부모 기준 레이어 계산 후 배치. `dir==='v'`(세로): 레이어를 위→아래로 층층이(각 레이어는 가로 한 줄), 엣지 `fromSide:'bottom'/toSide:'top'`. `dir==='h'`(가로): 좌→우, `right/left`. 끝에 `renderBoard(); fitView(); markDirty(); bToast`. forge 식별자 사용(`boardState`/`measure`/`renderBoard`/`fitView`/`W_NODE`/`bToast`/`markDirty`).
- 버튼: 헤더(또는 팔레트)에 `세로 정렬`·`가로 정렬` 2개. 클릭 → `autoLayout('v')`/`autoLayout('h')`.
- **기본 세로**: `seedDefaultStrategy()` 직후 `autoLayout('v')` 적용(boot 시드·newDoc 모두) → 새 포지는 세로 배치로 시작. (현재 seed는 가로 좌표 — 세로 정렬로 덮음.)

## 3. 비범위

- R2(대표이미지=차트)/R4(전문 블록)/R5(포지 분석 재생)/엘리어트/비전은 다음 사이클. 실데이터·이미지 분석 없음.

## 4. 검증

- 코어(forge-core.js) 무변경 → `node --test map/forge-core.test.js` 15/15 유지.
- 헤드리스: 사이드바 헤더 "포지"·"＋ New Forge", 새 포지 제목 "New Forge", 세로 정렬 버튼 동작(노드 위→아래 층 배치), 기본 시드가 세로로 표시. 콘솔 에러 0.
- 라이브 배포(forge.html). 서버/데이터 무관(forge-api.php 불변). 기존 map 파일 불가침.

## 5. 리스크/주의

- autoLayout가 `markDirty()` 호출 → 정렬도 서버 자동저장(정상). `parentsOf`가 사이클 그래프에서 무한루프 안 되게 `vis` 가드(map 알고리즘에 포함).
- 세로 정렬 후 엣지 side가 bottom/top으로 바뀜 → 직각 라우팅/오버레이 좌표 정합 확인.
- 리브랜드는 라벨만 — `DOCS`/서버 `documents` 키 변경 금지(기존 저장 데이터 호환).
- 단일 페이지·바닐라·noindex 유지. map.html/chart.html/api.php/map 데이터 불가침.

## 6. 다음 사이클 결정 기록 (R2/R4/R5)

- **R2**: 포지 대표 이미지가 우측 패널 가격차트로 크게 표시(현 합성 캔들 대체/병행). 라이브러리 드래그로 대표 이미지 교체. 가격차트 ≡ 대표 이미지.
- **R4**: 블록 팔레트에 **파동 스캔**(chart.html 주기 스캔, 추가 시 자동 스캔) + **엘리어트 파동** 추가. 각 도구가 전문 분석 수준.
- **R5**: **"포지 분석" 재생 버튼** — 노드 순서대로 분석 진행, 대표 이미지 위에 각 노드의 최적 보조선/스캔 표시, 예측 실시간 반영·수렴, 중요도/확신에 따라 백트래킹.
- **분석 데이터 원천**: Claude 비전 베스트-에포트 — 이미지에 데이터+축 있으면 정밀, 없으면 파동·추세 등 읽어 우측 패널 반영. → D(비전)가 앞당겨짐(마지막 아님).

## 7. 확정된 결정

1. R1 리브랜드는 UI 라벨만(내부/서버 키 불변).
2. R3 autoLayout('v'/'h') 포팅 + 버튼 2개 + 기본 세로(시드 후 autoLayout('v')).
3. R2/R4/R5는 다음 사이클(들), Claude 비전 베스트-에포트 기반.
