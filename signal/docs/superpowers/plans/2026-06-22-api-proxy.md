# cafe24 api.php 프록시 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** cafe24 PHP 프록시(`api.php`)로 FRED·beaconcha의 브라우저 CORS 차단을 우회해 매크로(유동성 축)·검증자 큐를 실데이터화하고, 프론트 로더 2개를 프록시 경유로 전환.

**Architecture:** `api.php`는 `?u=<업스트림 URL>`을 받아 호스트 화이트리스트(stlouisfed·beaconcha) 검사 후 서버 사이드 cURL로 받아 CORS 헤더 붙여 반환(FRED 키 서버 주입·파일 캐시). 프론트는 `proxied(url)` 헬퍼로 loadFred/loadBeacon을 프록시 경유로 호출, 실패 시 기존 FALLBACK 유지(그레이스풀). 점수 산식·UI·다른 로더 불변.

**Tech Stack:** PHP(cafe24, 무프레임워크) + 기존 단일 HTML/Vanilla JS. 로컬엔 PHP 런타임 없음 → api.php 실검증은 cafe24 배포 후 `curl`(컨트롤러). 프론트는 WSL 헤드리스로 그레이스풀 폴백 검증.

## Global Constraints

- 신규 파일은 `signal/api.php` 하나, 프론트 수정은 `signal/scoopsignal.html`만. 외부 의존성 추가 금지.

- WAF: api.php는 실행 PHP라 `<?php` 정상(정적 파일 한정 제약). 출력 JSON에 `<?` 리터럴 미포함.

- [ ] **Step 1: api.php 작성 (전체 내용)**

`signal/api.php`를 아래 내용으로 생성한다(verbatim). `$FRED_KEY`는 **빈 문자열 유지**(배포 시 업로드본에만 주입):

```php
<?php
/**
 * ScoopSignal CORS 프록시 — FRED + beaconcha 전용(화이트리스트).
 * 프론트(scoopsignal.html)가 동일 호스트 ./api.php?u=<업스트림 URL> 로 호출.
 * 브라우저 CORS·키 차단 우회. FRED 키는 이 서버 파일에만(클라이언트 미노출).
 * 주의: 화이트리스트 외 호스트 차단(오픈 프록시 아님). jsiy 등 다른 파일·데이터 미수정.
 */
header('Access-Control-Allow-Origin: *');
header('Content-Type: application/json; charset=utf-8');

$FRED_KEY = '';   // ← cafe24 업로드본에만 FRED 무료 키 주입(git엔 빈 슬롯)

$u = isset($_GET['u']) ? $_GET['u'] : '';
$p = @parse_url($u);
if (!$p || empty($p['scheme']) || !in_array(strtolower($p['scheme']), ['http','https']) || empty($p['host'])) {
  http_response_code(400); echo json_encode(['error' => 'bad url']); exit;
}
$host = strtolower($p['host']);
$path = isset($p['path']) ? $p['path'] : '';
$allow = ['api.stlouisfed.org' => '/fred/', 'beaconcha.in' => '/api/'];
if (!isset($allow[$host]) || strpos($path, $allow[$host]) !== 0) {
  http_response_code(403); echo json_encode(['error' => 'host not allowed']); exit;
}

// FRED 키 서버 주입(클라이언트 URL엔 키 없음)
$target = $u;
if ($host === 'api.stlouisfed.org' && $FRED_KEY !== '') {
  $target .= (strpos($u, '?') === false ? '?' : '&') . 'api_key=' . urlencode($FRED_KEY);
}

// 파일 캐시 (FRED 1h / beaconcha 10m)
$ttl = ($host === 'api.stlouisfed.org') ? 3600 : 600;
$cacheFile = __DIR__ . '/api_cache_' . md5($u) . '.json';
if (is_readable($cacheFile) && (time() - filemtime($cacheFile)) < $ttl) {
  echo file_get_contents($cacheFile); exit;
}

$ch = curl_init($target);
curl_setopt_array($ch, [
  CURLOPT_RETURNTRANSFER => true,
  CURLOPT_TIMEOUT        => 15,
  CURLOPT_FOLLOWLOCATION => false,
  CURLOPT_HTTPHEADER     => ['accept: application/json'],
  CURLOPT_USERAGENT      => 'ScoopSignal/1.0 (+moneyscoop.co.kr)',
]);
$resp = curl_exec($ch);
$code = (int) curl_getinfo($ch, CURLINFO_HTTP_CODE);
curl_close($ch);

if ($resp === false || $code < 200 || $code >= 300) {
  http_response_code($code ?: 502);
  echo json_encode(['error' => 'upstream', 'code' => $code]); exit;
}
@file_put_contents($cacheFile, $resp);
echo $resp;
```
