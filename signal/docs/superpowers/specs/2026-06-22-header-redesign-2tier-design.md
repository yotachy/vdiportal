# 헤더 재설계 — 2단 구조(유료 서비스 대비) 설계 문서

- 작성일: 2026-06-22
- 대상: `signal/scoopsignal.html` (단일 정적 HTML)
- 성격: 헤더 레이아웃·UX 재설계(프론트 시각/구조). 점수 산식·데이터 로더·차트 렌더·뷰 라우터 로직 불변.

## 1. 배경 / 목적

현재 헤더(`.top-bar`)는 `[브랜드+자산탭] … [업그레이드 + "불러오는 중…" + 새로고침]`을 한 베이스라인(`align-items:flex-end`)에 욱여넣어 위계가 없다. 브랜드 정체성과 제품 네비(자산 전환)가 붙고, 우측 유틸리티("불러오는 중…" 생텍스트 + 골드 버튼 + 아이콘)가 뒤섞인다. 좁은 폭에서 줄바꿈도 지저분하다.

이 서비스는 향후 **유료(회원·플랜)** 로 전환된다(Basic/Signature 데이터 게이팅 — [[프로젝트 CLAUDE.md]]). 따라서 헤더를 **브랜드 / 자산 네비 / 계정·플랜·유틸리티 3존**이 분명한 구조로 재설계하고, 계정·플랜·업그레이드 동선을 미리 확보한다(일부는 현 단계 플레이스홀더).

## 2. 목표 (사용자 확정)

- **2단 헤더**: Tier1=브랜드(좌)+유틸리티(우), Tier2=자산 네비바.
- 우측 유틸리티 4요소: **갱신시각+새로고침 / 플랜 배지(무료) / 로그인 슬롯 / 업그레이드 CTA**.
- **모바일 반응형 중점**: 좁은 화면에서 깨지지 않고, 계정·플랜은 드로어로 이동.

## 3. 비목표 (YAGNI)

- 실제 로그인/회원/결제 백엔드. (로그인은 준비중 슬롯, 플랜은 '무료' 정적 표시.)
- 자산 전환 실제 동작(비트코인/알트코인은 기존대로 `disabled` '준비중').
- 점수·차트·데이터 로직 변경. 사이드바 네비(좌측) 구조 변경(드로어 상단 계정 블록 추가만).

## 4. 구조 설계

```
Tier 1 (.hdr-top)   ☰  ◈ CryptoSignal · by MoneyScoop          ⟳ 마지막 갱신 09:41 · 자동   [무료]  로그인  [업그레이드]
──────────────────────────────────────────────────────────────────────────────────────────────
Tier 2 (.hdr-nav)     암호화폐 전체    비트코인    [이더리움]    알트코인
```

`.top-bar`(sticky)는 두 자식 `.hdr-top`(flex, space-between)과 `.hdr-nav`(자산 탭 스트립)를 세로로 쌓는다.

### 4.1 Tier 1 — `.hdr-top`

- **좌 `.hdr-brand`**: `☰`(`.hamb`, 모바일만) + 작은 브랜드 마크(◈, ETH 다이아 SVG, `--gold`/`--eth`) + `.wordmark`("CryptoSignal" gold em + "by MoneyScoop" 작게). 클릭/Enter → `showView('dashboard')`(기존 `#brandHome` 핸들러 유지). 마크도 같은 클릭 영역에 포함.
- **우 `.hdr-util`**(flex, align-center, gap):
  - **상태 `.hdr-status`**: ⟳ 새로고침 버튼(`#refreshBtn` 유지) + "마지막 갱신 HH:MM"(`#updatedAt`) + "· 자동"(faint). 기존 "불러오는 중…" 텍스트를 이 묶음으로 대체. `#updatedAt` id·갱신 로직 불변.
  - **플랜 배지 `.plan-badge`**: 텍스트 "무료"(틴트 칩, `--panel` 배경/`--muted`). Pro 전환 자리.
  - **로그인 `.login-btn`**: 고스트 버튼("로그인"), 지금은 `disabled` + `title="준비 중"`.
  - **업그레이드 `.upgrade-btn`**: 기존 골드 버튼 유지(`#upgradeBtn`).

### 4.2 Tier 2 — `.hdr-nav` (자산 네비바)

- 기존 `.asset-switch`(밑줄형 탭) 유지하되 전폭 스트립으로: `.hdr-nav{display:flex;border-bottom:1px solid var(--line)}` 안에 탭 배치. 활성 이더리움 골드 밑줄.
- 자산 메뉴명·`준비중` 칩·disabled 상태는 현행 유지(암호화폐 전체/비트코인/이더리움/알트코인).
- 기존 `.top-bar`의 하단 `border-bottom`은 Tier2 네비바 하단으로 이전(이중선 방지).

### 4.3 태그라인 제거

- 기존 `.tagline`("이더리움 신호 엔진 · 가격·온체인·유동성·사이클 4축 점수화")는 헤더에서 **제거**. (다자산 네비와 충돌 + 대시보드 본문이 맥락 제공.) 관련 CSS(`.tagline`, `.eth-logo`)도 정리.

## 5. 반응형 설계 (중점)

- **≥901px (데스크톱)**: Tier1 전체(상태 텍스트 포함) + Tier2 노출. `.hamb` 숨김.
- **≤900px (태블릿·모바일·폴드)**:
  - `.hamb` 노출. Tier1 우측 `.hdr-util`은 **`[⟳][업그레이드]`만** 노출(컴팩트). `.hdr-status`의 텍스트("마지막 갱신…·자동")는 숨기고 ⟳ 아이콘만.
  - **플랜 배지·로그인·갱신시각 텍스트 → 햄버거 드로어 상단 계정 블록(`.side-acct`)으로 이동**. 드로어(`.side`)의 닫기 버튼(`#sideClose`) 아래, `.snav` 위에 `.side-acct` 추가: "무료" 배지 + "로그인"(준비중) + "마지막 갱신 HH:MM". (드로어에서만 표시, 데스크톱 숨김.)
  - Tier2 자산탭: `overflow-x:auto`(4개라 대개 한 줄, 안전장치). 고정 높이로 줄바꿈 깨짐 방지.
- **≤560px**: 브랜드 "by MoneyScoop" 숨김(`.wordmark .by{display:none}`), 업그레이드 버튼 축약(아이콘+짧은 라벨 또는 패딩 축소), 탭 폰트·gap 축소. Tier1 고정 높이·`flex-wrap:nowrap`로 1줄 유지, 넘치면 우측 클러스터 우선순위(⟳·업그레이드)만.

**계정 블록 동기화**: `#updatedAt`은 데스크톱 상태 묶음과 드로어 계정 블록 양쪽에 같은 값이 필요. 한 곳(`#updatedAt`)만 유지하고, 드로어용은 **CSS로 위치만 다르게** 하거나 별 id(`#updatedAtSide`)에 동일 텍스트를 기존 갱신 함수가 함께 써넣는다. → 갱신 함수가 두 곳을 갱신하도록 최소 수정(렌더 대상만 추가, 로직 불변).

## 6. 비주얼 / 토큰

- 디자인 토큰만(색 하드코딩 금지). "박스 남발 금지" 원칙 준수: 유틸리티 영역은 면+여백으로 구분, 보더는 버튼(`.login-btn`·`.upgrade-btn`)·배지(`.plan-badge`)·네비바 하단선에만.
- `html{zoom:1.35}` 유지, 한국어, letter-spacing 관행, 좌측 컬러바 금지(마크는 SVG 면), 2 spaces.
- 브랜드 마크: 인라인 SVG(ETH 다이아, 기존 `.eth-logo` path 재활용 가능), `viewBox="0 0 24 24"` 또는 32, 16~18px.

## 7. 엣지 / 주의

- `#brandHome`·`#refreshBtn`·`#upgradeBtn`·`#updatedAt` id와 그 핸들러/갱신 로직 유지(마크업 재배치만). `#sideClose`·`openSide`·`showView` 불변.
- sticky 헤더 높이 변동 → `--top-bar-h`(현 104px) 재산정. 2단 실제 높이에 맞춰 토큰 갱신(사이드바 sticky top 계산에 사용됨). 데스크톱/모바일 높이 차 주의.
- 드로어 계정 블록은 ≤900px에서만 보이고 데스크톱 숨김. 중복 마크업 최소화(갱신값은 함수가 양쪽 기록).
- 줄바꿈/overflow: Tier1 `flex-wrap:nowrap` + 우측 클러스터 축소 우선순위로 깨짐 방지.

## 8. 검증

- **헤드리스 스크린샷** 3폭: 1280 / 768 / 390.
  - 1280: 2단 노출, 우측 4요소 정렬, 활성 이더리움 밑줄, 상태 묶음 텍스트.
  - 768·390: ☰ 노출, 우측 `[⟳][업그레이드]`만, 드로어 열어 계정 블록(무료·로그인·갱신시각) 표시, 자산탭 1줄·깨짐 없음, Tier1 1줄 유지.
- 기능: 타이틀/마크 클릭→대시보드, 새로고침 동작, 드로어 열기/닫기/계정 블록, `#updatedAt` 양쪽 동기화.
- JS 에러 0, 점수·차트 회귀 0(렌더 대상 id·함수명 불변).
- `--top-bar-h` 갱신 후 사이드바 sticky 위치 정상(콘텐츠와 안 겹침).
- 육안: 위계 명확(브랜드/네비/유틸 분리), "엉망" 해소.
