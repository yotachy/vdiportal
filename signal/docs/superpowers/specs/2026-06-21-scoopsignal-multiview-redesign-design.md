# ScoopSignal 멀티뷰 리디자인 — 설계 문서

- 작성일: 2026-06-21
- 대상 파일: `signal/scoopsignal.html` (단일 정적 HTML 유지)
- 성격: 구조·디자인 리팩터 (데이터/산식 로직 불변)

## 1. 배경 / 문제

현재 `scoopsignal.html`은 단일 뷰다. 게이지·레이더·사이클시계·4축 카드가 항상 보이고, 그 아래 주기 패턴 7종(계절성·사이클오버레이·반감기·로그밴드·200주배수·드로다운·변동성)이 **하나의 탭 박스**에 좁게 몰려 있다. 또한:

1. **금지 디자인 사용 중** — 축 카드의 `.card::before`(좌측 3px 컬러바)와 사이드바 활성 항목의 `box-shadow:inset 3px 0 0 var(--gold)`(좌측 골드 바). 사용자 지침상 "왼쪽 바 형태 색상 적용" 방식은 절대 금지.
2. **브랜드가 사이드바에 묻혀 있음** — 스쿱시그널은 앞으로 전체·BTC·ETH·알트 등 여러 신호 엔진으로 확장된다. "지금 보는 것이 ETH 에디션"임을 인지시킬 **표준 헤더**가 없다.
3. **패턴이 전용 페이지가 아님** — 좁은 탭 박스라 차트가 작고 비교/판독이 불편하다.

## 2. 목표

- 단일 HTML 파일 제약을 유지하면서 **JS 뷰 전환(SPA)** 으로 다중 "전용 페이지" 구현.
- 모든 자산 에디션이 공유할 **표준 상단 헤더** 도입 + 자산 스위쳐로 ETH 에디션임을 인지.
- 금지된 좌측 컬러바를 **은은한 전체 틴트**로 대체.
- 데이터 로더·점수 산식·`recompute()` 계산은 **불변**(렌더 타깃/가시성만 변경).

## 3. 비목표 (YAGNI)

- BTC·알트·전체 에디션의 실제 신호 엔진 구현 (스위쳐는 ETH만 활성, 나머지 `준비중` 비활성 자리표시).
- 점수 산식·임계값·데이터 소스 변경.
- 파일 분리(여전히 단일 `scoopsignal.html`).
- 라우팅 해시·딥링크(필요 시 후속). 1차는 메모리 상태 전환으로 충분.

## 5. 컴포넌트 설계

### 5.1 표준 상단 헤더 (재사용 템플릿)

전 자산 에디션이 공유하는 구조. ETH 고유 값은 텍스트만 교체하면 됨.

- `머니스쿱` 아이브로우(eyebrow, 골드).
- `ScoopSignal` 워드마크 (`시그널` 골드 강조 유지).
- **자산 스위쳐**: 세그먼티드 컨트롤 `전체 · 비트코인 · 이더리움 · 알트`.
  - `이더리움`만 활성: 골드 배경/텍스트 + `aria-current="page"`.
  - 나머지 3개: `disabled` + `title="준비 중"` + 흐린 색. 클릭 시 `showToast("준비 중인 에디션입니다")`.
  - 자산별 메타는 JS 객체 `ASSETS`로 관리(현재 ETH만 `live:true`).
- 표준 태그라인: `이더리움 신호 엔진 · 가격·온체인·유동성·사이클 4축 점수화`.
- 우측 액션: 새로고침 버튼 + `업데이트 HH:MM:SS`(기존 `#updatedAt` 이전).
- sticky(`top:0`), 하단 1px 라인.

### 5.2 사이드바 (네비 전용으로 축소)

- 브랜드 블록 제거(헤더로 이전). 네비 + 데이터 상태 푸터만 남김.
- 네비 항목 = `data-view` 라우팅 키:
  - `dashboard` (그룹 없음, 최상단)
  - 그룹 **사이클**: `season` · `cycle` · `halving`
  - 그룹 **밸류·리스크**: `band` · `mayer` · `dd` · `vol`
- 각 항목 우측 **현재값 배지**(`updateSideBadges`) 유지. 대시보드 항목은 배지 없음(또는 종합점수).
- **활성 표현(좌측 바 금지)**: 배경 `--panel-2` + 테두리 `--line` + 텍스트 골드 + 선행 점(`•`). `box-shadow:inset 3px 0` 제거.
- 반응형: 좁은 화면에서 사이드바가 상단 그리드 네비로 전환(기존 `@media(max-width:900px)` 패턴 유지).

### 5.3 메인 — 8개 뷰

각 뷰는 `<section data-view="{key}">`. 라우터가 한 개만 표시.

| key | 뷰 | 구성 |
|---|---|---|
| `dashboard` | 대시보드 | 게이지·레이더·사이클시계(hero) + 4축 카드 + 튜닝 아코디언 + 산식 아코디언 |
| `season` | 계절성 | 페이지헤더 + 월별 히트맵(크게) |
| `cycle` | 사이클 오버레이 | 페이지헤더 + `cvCycle` |
| `halving` | 반감기 | 페이지헤더 + `cvHalving` |
| `band` | 로그 밴드 | 페이지헤더 + `cvBand` + `#bandStats` |
| `mayer` | 200주 배수 | 페이지헤더 + `cvMayer` + `#mayerStats` |
| `dd` | 드로다운 | 페이지헤더 + `cvDD` + `#ddStats` |
| `vol` | 변동성 | 페이지헤더 + `cvVol` + `#volStats` |

- **티커**(ETH/BTC/ETH·BTC)는 모든 뷰 상단에 공통 표시(전역 컨텍스트).
- 패턴 뷰 **페이지 헤더**: 제목 + 한 줄 설명(기존 `chart-note` 문구 재사용) + 현재값 리드아웃(`#*Stats`).
- 패턴 **차트 높이 300→420px**(전용 페이지 판독성). `canvas{height}` 뷰별 조정.

### 5.4 축 카드 — 틴트 신호(좌측 바 대체)

- `.card::before`(좌측 3px) **삭제**.
- `data-sig`별:
  - 면 전체 배경: 신호색 5% 틴트(예: `--bull-dim`보다 옅게, 별도 `--*-tint` 토큰 또는 기존 dim 위에 조정).
  - 테두리: 신호색 ~30%.
- 기존 칩(`.chip`)·분위 막대(`prMeter`)는 유지(좌측 바와 무관).

## 6. 데이터/로직 흐름 (불변)

- `loadBinance/loadLlama/loadDollar/loadFred/loadBeacon` → `S` 저장: **변경 없음**.
- `scoreMom/scoreLiq/scoreFun/scoreVal`, `recompute()`, 튜닝/`CFG`/localStorage: **변경 없음**.
- `recompute()`는 계속 게이지·레이더·사이클시계·4축 카드(대시보드 타깃)와 `updateSideBadges()`를 갱신하고, **현재 활성 뷰가 패턴이면 그 차트를 그린다**.

## 7. 뷰 라우터 (탭 로직 교체)

기존 `selectPattern`/`activeTab`/`.snav-item[data-tab]`/`.panel-chart` 구조를 **뷰 라우터**로 일반화.

- 상태: `let activeView='dashboard';`
- `showView(key)`:
  1. `.snav-item`의 `.on` 토글(`data-view===key`).
  2. `<section data-view>` 표시 토글.
  3. 패턴 뷰면 해당 draw 함수 호출(데이터 준비 시). 매핑: `{season:drawHeatmap, cycle:drawCycle, halving:drawHalving, band:drawBand, mayer:drawMayer, dd:drawDrawdown, vol:drawVol}`.
- `recompute()` 끝에서 `activeView`가 패턴이면 해당 draw 호출(기존 `drawActiveChart` 일반화).
- resize 디바운스: 활성 뷰가 캔버스 차트면 재draw(기존 로직 일반화).
- 기본 진입 뷰: `dashboard`.

## 8. 엣지/주의

- **캔버스 가시성**: 숨겨진 섹션에서 `clientWidth/Height`가 0이면 차트가 깨짐 → 반드시 **표시 후 draw**(지연 렌더 유지). `display:none` 토글 방식 사용.
- **첫 로드**: 데이터 도착 전 패턴 뷰로 들어가도 안전(빈 차트/`데이터 부족` 가드 기존 유지).
- **자산 스위쳐 비활성 클릭**: 라우팅 아님 → 토스트만. `showToast`가 없으면 간단 생성(현 파일엔 토스트 없음 → 최소 토스트 유틸 추가).
- **반응형 헤더**: 자산 스위쳐 + 액션이 좁은 폭에서 wrap.
- **zoom 1.35 유지**, 디자인 토큰만 사용, 하드코딩 색 금지(필요 시 `--*-tint` 토큰 추가).

## 9. 변경 범위 요약

- HTML: 헤더 신설, 사이드바 축소(네비/상태만), 메인을 8개 `data-view` 섹션으로 재배치, 패턴별 페이지 헤더 추가.
- CSS: 헤더/자산 스위쳐 스타일, 카드 틴트 신호, 활성 네비(좌측 바 제거), 패턴 차트 높이, 반응형 보정. 죽은 탭 CSS 제거.
- JS: `selectPattern`→`showView` 라우터, `drawActiveChart`→`activeView` 기반, 자산 스위쳐 바인딩, 최소 `showToast` 유틸. **데이터·산식 불변.**

## 10. 검증

- 8개 뷰 전환 시 해당 차트가 정상 크기로 렌더되는지(특히 캔버스 4종).
- 60초 자동 새로고침 후에도 활성 뷰 유지·갱신.
- 좌측 컬러바가 카드·네비 어디에도 없는지(금지 준수).
- 자산 스위쳐: ETH 활성, 나머지 비활성+토스트.
- 튜닝 패널 조작 시 네트워크 없이 즉시 반영(기존 동작 보존).
- 헤드리스 스크린샷(WSL playwright)으로 대시보드 + 패턴 1뷰 시각 확인.
