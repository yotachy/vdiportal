# 대시보드 과감한 개편 — 조밀 지표 그리드 설계 문서

- 작성일: 2026-06-22
- 대상: `signal/scoopsignal.html` (단일 정적 HTML)
- 성격: 대시보드 랜딩 정보구조(IA)·시각 전면 개편 + 폰트 통일 + 새로고침 제거. 점수 산식·데이터 로더·14개 상세 뷰 로직 불변.

## 1. 배경 / 목적

현재 대시보드 랜딩은 데이터(사이드바에 14개 지표 라이브)가 풍부한데도 **큰 카드 몇 개**(거대 시그널 게이지 + 4줄 레이더 + 사이클 "분배" 한 단어)만 떠 있어 비어 보이고, 티커가 공간을 과하게 먹으며, 자산 탭 대비가 약하다. 유료는커녕 무료 기준으로도 "빈약한" 인상. 목표: **전문 암호화폐 지표분석 사이트(Glassnode·CoinGecko·Messari) 같은 조밀한 지표 대시보드**로 개편.

## 2. 목표 (사용자 확정)

- **조밀 지표 그리드**: 사이드바 14개 지표를 랜딩으로 끌어올려 한 화면에 많은 정보.
- **컴팩트 시그널 게이지 카드 + 레이더 나란히**(현재 절반 크기, 시각 앵커 유지).
- **또렷한 세그먼트 자산 탭 + 활성 강조**.
- KPI 타일에 **미니 스파크라인**(히스토리 가용 지표).
- **새로고침 버튼 완전 제거**(자동 120초 + "갱신 HH:MM · 자동" 표기 유지).
- **폰트 정책 통일**(타입 스케일 토큰화 후 전면 적용).

## 3. 비목표 (YAGNI)

- 점수 산식·데이터 로더·14개 상세 뷰 변경. 새 데이터 소스. 실시간 가격 스트리밍.
- KPI 타일 클릭 시 새 화면(기존 상세 뷰 `showView`로 재활용).
- 자산 전환 실제 동작(비트코인/알트코인 여전히 준비중).

## 4. 데이터 — 기존 캐시 재사용 (재fetch 없음)

`updateSideBadges()`가 이미 14개 지표의 현재값+호악색(`bull/bear/muted`)을 계산한다(season/spiral/cycle/halving/band/mayer/dd/vol/supply/gas/staking/defidom/treasury/trigger/squeeze/etf). **이 계산을 단일 출처로 리팩터**하여 사이드바 배지와 새 KPI 타일이 같은 값을 쓴다(DRY).

- 신규 `S._metrics` 맵: `{ key: {val:string, sig:'bull'|'bear'|'muted', sub?:string} }`. `updateSideBadges()`를 `buildMetrics()`(맵 생성) + 맵 소비(배지 set)로 분리. KPI 그리드도 같은 맵 소비.
- **스파크라인 시계열**(가용 시): `S.ma200.hist`(200주배수), `S.us.series`(순발행·공급), `S.week.c`(드로다운·로그밴드·변동성 파생), `S.month.c`(계절성). 없는 지표(가스·스테이킹·DeFi·기관·트리거·공급압력·계절성 등 단일값)는 스파크라인 생략 → 값+부가만(그레이스풀).
- 시그널 점수·4축·판정·사이클 위상: 기존 `recompute()` 산출물(`S.score`, 축점수, verdict, cycle phase) 재사용.

## 5. 레이아웃 설계 (대시보드 뷰)

기존 `data-view="dashboard"` 섹션 본문을 아래로 교체. 헤더(2단)·사이드바·14개 상세 뷰는 그대로.

### 5.1 압축 스탯바 `.dash-stats`

- 1행, 조밀(현재 큰 티커 밴드 대체). 셀: **ETH/USD + 등락% · ETH/BTC + 등락% · 도미넌스 · 종합점수+판정**.
- 값=mono, 라벨=sans muted. 큰 박스 없이 면+얇은 구분(틴트 그룹 원칙). 모바일 2열 wrap.

### 5.2 앵커 행 `.dash-anchor`

- 좌: **컴팩트 시그널 게이지 카드**(`.sig-card`) — 반원 게이지(현재 폭 ~절반), 점수(mono ~30), 판정(약세/중립/강세 색), 1줄 설명, 사이클 위상 소형 칩. 클릭→`signal`.
- 우: **레이더 4축 컴팩트 카드** — 4축 막대(유동성/모멘텀/펀더멘털/밸류, 값+분위 색). 클릭→`radar`.
- 2열(데스크톱) → 1열(모바일).

### 5.3 KPI 지표 그리드 `.kpi-grid` (핵심)

- 3그룹(사이드바 분류 일치), 각 그룹 소제목(`.kpi-group`, 10px uppercase tracked) + 타일 행:
  - **밸류·리스크**: `mayer`(200주배수) · `band`(로그밴드 z) · `dd`(드로다운) · `vol`(변동성)
  - **온체인·네트워크**: `supply`(순발행·공급) · `gas`(가스) · `staking`(스테이킹) · `defidom`(DeFi지배력)
  - **사이클·기관**: `cycle`(사이클 주차) · `season`(계절성) · `treasury`(기관보유) · `squeeze`(공급압력) · `trigger`(트리거)
- **타일 `.kpi`**: `라벨(sans muted) · 값(mono, 호악색) · 부가/변화(작게) · 미니 스파크라인(가용 시, 호악색 라인) · 우상단 ›`. 전체가 버튼 → `showView(key)`. 박스 남발 금지(약한 틴트 면 + 라운드 외곽, 보더 없음 / hover 시 inset 골드 링).
- 반응형: 4열(≥1024) → 2열(≤880) → 1~2열(≤520). 그룹 소제목은 유지.
- 스파크라인: 공용 `sparkline(canvas, series, sig)` (기존 `lineChart` 축약형 또는 신규 경량 함수). 높이 ~28px, 축·라벨 없음.

### 5.4 하단

- 튜닝·방법론 `.toolbar` 토글 + 면책 유지(위치만 그리드 아래).

## 6. 자산 탭 개편 `.hdr-nav`

- 또렷한 세그먼트: 탭 폰트 ↑(13px), 활성(이더리움)=골드 텍스트+굵게+하단 골드 바(현행 강화) 또는 골드 soft 채움 칩. 비활성 가독 색(`--muted` 수준, 현재 `--faint`보다 또렷). 준비중 3개는 `준비중` 칩을 더 작고 흐리게(라벨 자체는 또렷). 대비 강화로 "거의 안 보임" 해소.

## 7. 폰트 정책 통일 (타입 스케일)

- 숫자=`--mono`(JetBrains Mono), 텍스트=`--sans`(Pretendard) 일관.
- 스케일(zoom 1.35 환경, CSS px): 게이지 점수 ~30 / 스탯값 ~18 / 타일값 ~17 / 페이지·카드 제목 ~14 / 타일·스탯 라벨 11.5(muted) / 그룹 소제목 10(uppercase, letter-spacing .16em) / 본문 12.5 / 캡션 11.
- letter-spacing: 제목 -.02em, 본문 -.01em. 카드마다 제각각인 크기·굵기 제거. 기존 `.sec-head`/`.page-head`/타일/스탯이 같은 스케일을 참조하도록 정리.

## 8. 새로고침 제거

- 헤더 `.hdr-status`의 `#refreshBtn`, 드로어 `.side-acct`의 `.refresh-ic` **둘 다 제거**.
- `refresh()`의 `.refresh-ic` 토글/바인딩은 빈 NodeList여도 무해(자동 `setInterval(refresh,120000)` 유지). "갱신 HH:MM · 자동"(`#updatedAt`/`#updatedAtSide`) 표기는 유지.

## 9. 그레이스풀 / 검증 환경

- 데이터 미연결(Binance/Upbit 지오블록, 일부 소스 실패)이어도 가용 지표만 채우고 나머지는 "—"(`buildMetrics` 기존 가드 유지). 스파크라인은 시계열 없으면 생략.
- 모든 KPI 타일·스탯은 `S` 캐시에서 렌더, `recompute()` 흐름에서 호출(재fetch 없음).

## 10. 검증

- **헤드리스 스크린샷** 3폭: 1280 / 768 / 390.
  - 스탯바 1행 조밀, 시그널+레이더 컴팩트 나란히, KPI 그리드 3그룹·타일 다수 표시(값·색·가용 스파크라인), 빈 공간 해소.
  - 자산 탭 또렷·이더리움 활성 강조.
  - 새로고침 버튼 헤더·드로어 모두 없음(`#refreshBtn` 부재), 자동 갱신 표기 존재.
  - 폰트 스케일 일관(타일값·스탯값 mono, 라벨 sans).
  - KPI 타일 클릭 → 해당 상세 뷰 전환.
- 반응형: 1280 4열 / 768 2열 / 390 1~2열, 깨짐 0.
- JS 에러 0, 점수·차트·상세 뷰 회귀 0(렌더 대상 id·함수명·`showView`/`VIEW_DRAW` 불변).
- 사이드바 배지(기존)도 `buildMetrics` 리팩터 후 동일 값 유지(회귀 0).
- 육안: "빈약" 해소, 전문 분석 사이트 밀도·일관 타입.

## 11. 엣지 / 주의

- `buildMetrics` 리팩터 시 기존 사이드바 배지 값·색이 100% 동일해야 함(로직 이전만, 수치 불변).
- KPI 타일은 `recompute()`에서 `renderKpiGrid()` 호출(시그널·레이더 렌더 뒤). 캐시만 사용.
- 박스 남발 금지: 타일은 약한 틴트 면(보더 없음), 그룹은 소제목+여백으로 구분.
- 새로고침 제거 후 `refresh()`가 버튼 없이도 정상(자동 갱신·수동은 없음) — 사용자 확정.
- KPI 그리드가 길어지면 사이드바보다 길어질 수 있음(데스크톱 `.side` sticky·콘텐츠 높이) — 정상(콘텐츠 우선).
- 스파크라인 canvas는 DPR·줌 고려해 선명하게(기존 lineChart의 DPR 처리 패턴 따름).
