# 박스 남발 제거 — 틴트 그룹 + 상단 크롬 접기 설계 문서

- 작성일: 2026-06-22
- 대상: `signal/scoopsignal.html` (단일 정적 HTML) + `signal/CLAUDE.md`(지침 1줄 추가)
- 성격: **순수 시각 정리(디자인 품질)**. 점수 산식·데이터 로더·차트 로직·렌더 함수 동작 불변.

## 1. 배경 / 목적

라운드+보더 카드 박스가 `ticker / pane / card / hero-tile / status / method / tune / side / trig` 등 여러 겹으로 중첩되어 "카드 수프"가 됨 → 디자인 품질 저하. 특히 **콘텐츠 차트 뷰 자체는 이미 깔끔**(`.page-head` + 캔버스)한데, 그 위·주변을 박스가 둘러싸 산만함을 만든다.

사용자 확정 방향:
- **디자인 언어 = 은은한 틴트 그룹**: 보더 제거 → 배경 틴트 차이(`--ink`→`--panel`→`--panel-2`)와 여백으로 그룹핑. 라운드는 최외곽 컨테이너에만.
- **상단 크롬 = 접기+한 줄**: 방법론·튜닝은 한 줄 텍스트 토글, 데이터 상태는 사이드바 하단으로. → 콘텐츠 위 박스 0개.

## 2. 디자인 원칙 (신규 — `signal/CLAUDE.md` §8에 추가)

> **카드(보더+배경+라운드) 박스를 남발하지 않는다.** 콘텐츠는 기본 "면(面)"으로 두고, 구분이 필요할 때만 **배경 틴트 차이**(`--ink`→`--panel`→`--panel-2`)와 여백으로 그룹핑한다. 보더(`--line`)는 외곽 1겹 또는 입력/버튼에만 쓴다. **라운드는 최외곽 컨테이너에만.** 박스 안의 박스(중첩 카드)는 금지. 카드 형태(보더+배경+라운드)는 **클릭 가능한 타일**에만 허용한다.

## 3. 토큰 (기존 그대로 사용)

`--ink:#0B0F14`(페이지 베이스) < `--panel:#141A22` < `--panel-2:#1B232D` < `--line:#232D38`. 새 토큰 도입 없음. 틴트 그룹은 이 스케일의 배경 차이로만 표현.

## 4. 변경 상세

### 4.1 상단 크롬 — 콘텐츠 위 박스 0개

- **데이터 상태(`.status`)**: 메인 콘텐츠 영역에서 제거 → **사이드바 하단**으로 이동. 보더·배경 없는 점-목록(이미 `@media(max-width:900px)`에 `.status{border:none;background:none;...}` 변형이 존재 — 이를 데스크톱에도 적용하고 사이드바 안에 배치). `#st-*` 상태 점 id·`setStatus()` 동작 불변.
- **방법론(`.method`)**: 박스(보더+배경+라운드) 제거 → 콘텐츠 상단 **한 줄 텍스트 토글**("방법론 ▾"). `<details>`/토글 구조 유지하되 summary를 보더 없는 텍스트 링크로. 펼친 본문은 약한 틴트(`--panel`) 배경, 보더 없음.
- **튜닝(`.tune`)**: 박스 제거 → 한 줄 텍스트 토글("⚙ 튜닝 ▾", **골드 색**으로 발견성 유지 — 점수에 영향하므로). 펼친 패널은 은은한 틴트, 보더 없음. 기존 골드 박스+글로우(`box-shadow`) 제거. 슬라이더·`bindTune`·`syncTuneUI`·`recompute()` 호출 동작 불변.
- 두 토글은 콘텐츠 최상단에 한 행으로 나란히(텍스트 링크 2개). 펼치면 그 자리에서 인라인 확장.

### 4.2 틴트 그룹 — 보더 제거, 라운드 외곽만

- **티커(`.ticker`)**: 외곽 보더 제거, 셀 구분 1px 라인 제거 → 은은한 틴트 스트립(`--panel` 배경, 라운드 외곽만). 셀(`.tk`)은 배경 없이 여백으로 구분.
- **사이드바(`.side`, 데스크톱 >900px)**: 패널 보더·배경 제거 → 페이지 위에 떠 있는 네비. 항목(`.snav-item`) hover/active만 틴트(`--panel-2`). 그룹 라벨·간격 유지. **모바일 드로어(≤900px)는 오버레이이므로 배경(`--panel`) 유지**(가독성).
- **시그널 게이지 면(`.pane.signal-pane`)**: 보더 제거 → 약한 틴트(`--panel`), 라운드 외곽만.
- **히어로 타일(`.hero-tile`)**: 클릭 가능한 타일이므로 카드 형태 허용하되 보더 제거 → 약한 틴트 + hover lift(약한 배경 변화). 라운드 유지.
- **4축 상세(`.card` ×4, radar 뷰)**: 보더 제거 → 보더 없는 틴트 블록 2×2. 라운드는 개별 블록 약하게(틴트로 구분되므로 보더 불요).
- **gas/staking 뷰**: `<div class="card" ...>` 래퍼 제거 → 스탯 블록을 플랫으로(보더·배경 없이 콘텐츠만). `#gasStats`/`#stakingStats` 렌더 대상 id 유지.
- **트리거 보드(`.trig`)**: 채움 박스(`--panel-2` 풀필) 행 → 가벼운 틴트 행(보더 없음, 상태 점 + 텍스트). 라운드 약하게 또는 제거.
- **`.card` 클래스 재정의**: 기본 보더 제거 + 약한 틴트(`--panel`). 라운드는 유지하되 중첩 사용 금지(원칙 2). (영향 받는 모든 `.card` 사용처가 틴트 블록으로 일관 전환.)

### 4.3 영향 없는 것 (불변)

- 점수 산식(`scoreMom/Liq/Fun/Val`, `recompute`, `normW`), 데이터 로더(`loadUpbit/Llama/Dollar/Fred/Beacon/Etf`), 차트 렌더(`lineChart`/`buildGauge`/`renderRadar`/`renderQuad`/`drawCycle/Band/Heatmap`), 뷰 라우터(`showView`/`VIEW_DRAW`/`activeView`), 티어 배지(`CHART_TIER`/`applyTierBadges`).
- `html{zoom:1.35}`, 한국어 UI, 디자인 토큰(색 하드코딩 금지), 2 spaces, 좌측 컬러바 금지.
- 햄버거 드로어 동작(`openSide`/백드롭/ESC/리사이즈) 불변 — 드로어 열림 시 사이드 배경 유지가 전제이므로 ≤900px `.side` 배경은 남긴다.

## 5. 그레이스풀 / 검증 환경

- 시각 전용 변경이라 데이터 소스 연결 여부와 무관(로컬 헤드리스 가능). Binance 지오블록·etf.php 미배포 상태에서도 레이아웃 검증 가능.
- 콘텐츠 데이터가 비어도(예: ETF "연결 필요") 박스 정리는 적용되어야 함.

## 6. 검증

- **헤드리스 스크린샷** 2종: 데스크톱 1280px / 모바일 390px.
  - 콘텐츠 영역 위에 보더 박스 0개(상태/방법론/튜닝 박스 사라짐).
  - 방법론·튜닝 한 줄 토글 클릭 시 인라인 확장·재접힘.
  - 데이터 상태가 사이드바 하단에 점-목록으로 표시.
  - 데스크톱 사이드바 무패널(보더·배경 없음), 항목 hover 틴트.
  - 모바일 ☰ 드로어는 배경 유지·정상 동작(기존 회귀 없음).
  - 시그널/4축/트리거/gas/staking 뷰에서 중첩 보더 박스 사라지고 틴트로 구분.
- **JS 에러 0**, 점수·차트 회귀 0(렌더 대상 id·함수명 불변 확인).
- 육안: "박스 수프" 해소, 에디토리얼하게 정돈된 인상.

## 7. 엣지/주의

- `.status`를 사이드바로 옮길 때 데스크톱/모바일 양쪽에서 한 번만 렌더(중복 마크업 금지). 기존 `#st-*` id 유일성 유지.
- 튜닝 토글이 접혀 있어도 슬라이더 값은 `CFG`(localStorage)에서 복원되어 점수에 반영됨(접힘=비활성 아님).
- 보더 제거로 인접 면 경계가 약해지는 곳은 여백(gap/margin)으로 분리. 라인 의존 금지.
- `.card`의 `overflow:hidden`(reveal/배지 stacking) 등 기능성 속성은 유지, 시각 속성(border/bg/radius)만 조정.
