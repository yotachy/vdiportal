# 메뉴 재구성(BitBo 참고) — 상단·사이드 + 프리미엄 플랫 아이콘 설계 문서

- 작성일: 2026-06-21
- 대상 파일: `signal/scoopsignal.html` (단일 정적 HTML 유지)
- 참고: BitBo(bitbo.io) — 효율적 사이드 카탈로그 + 최소 상단 + 유료(Upgrade) 모델
- 성격: 상단/사이드 메뉴 구조·시각 개선. 데이터/산식/뷰 라우터 불변, 실제 결제 없음.

## 1. 배경 / 목적

BitBo의 사이드바는 아이콘+라벨 조밀 리스트로 매우 효율적이다(다만 너무 작다). 크립토시그널도 차트가 계속 늘어날 예정이라, **아이콘 기반 효율 카탈로그**로 사이드바를 정돈하고 상단은 자산 스위쳐 중심의 최소 구성 + 향후 유료 진입점(업그레이드)을 둔다. 아이콘은 **통일감 있고 고급스러운 플랫(듀오톤)** 세트로 한다.

## 2. 목표

- **상단 바(최소)**: 워드마크 + ETH 태그라인 + 자산 스위쳐 유지 + 우측 **업그레이드 버튼** 신설(향후 유료, 지금은 토스트). 섹션탭 없음.
- **사이드바**: 주제 그룹·현재값 배지 유지 + 각 항목에 **프리미엄 플랫 듀오톤 아이콘** 추가. 적절한(여유있는) 행 높이.
- **Signature 표시 일원화**: 기존 네비 골드 '점' 제거 → Signature 항목 **아이콘을 골드**로(페이지 헤더 `◆ Signature` 배지는 유지).
- 아이콘은 외부 라이브러리 없이 **인라인 SVG**, 24×24 동일 그리드, 동일 구성 규칙으로 통일.

## 3. 비목표 (YAGNI)

- 실제 결제·로그인·요금제 페이지(업그레이드는 토스트만).
- 사이드바 접기(아이콘 전용) 토글.
- BitBo식 상단 섹션탭(Charts/Reports/Alerts/Terminal 등).
- 데이터 로더·점수 산식·뷰 라우터 변경.

## 4. 컴포넌트 설계

### 4.1 상단 바 — 업그레이드 버튼 추가

현재 `.tb-right`(업데이트 시각 + 새로고침 아이콘) 앞에 **업그레이드 버튼** 추가:
- 마크업: `<button class="upgrade-btn" id="upgradeBtn">…왕관 아이콘… 업그레이드</button>`.
- 스타일(토큰만): 골드 강조 — `color:var(--ink); background:var(--gold);` 라운드 필, 작은 왕관/별 SVG(16px) + 텍스트.
- 동작: 클릭 시 `showToast('요금제는 준비 중입니다')`. 실제 게이팅 없음.
- 반응형(≤560px): `.tb-right`가 이미 wrap → 업그레이드 버튼도 함께 줄바꿈, 크기 축소.

### 4.2 사이드바 — 아이콘 + 그룹/배지 유지

- 각 `.snav-item` 라벨 앞에 `<span class="snav-ic">…SVG…</span>` 삽입.
- 레이아웃: `[아이콘] 라벨 ……… [현재값 배지]`. 아이콘+라벨은 좌측 묶음, 배지는 우측(기존 `justify-content:space-between` 유지하되 좌측을 `display:inline-flex;gap` 묶음으로).
- `.snav-ic{width:17px;height:17px;flex:0 0 auto;display:inline-flex}` — 색은 `currentColor` 상속(항목 색을 따라감).
- 행 높이: 기존 `padding:9px 10px` 유지(적절한 크기). 주제 그룹 헤더·현재값 배지·데이터 상태 푸터 모두 유지.
- 활성(`.snav-item.on`): 골드 텍스트 → 아이콘도 골드 듀오톤 자동.

### 4.3 Signature 표시 일원화

- `applyTierBadges()`에서 **`.snav-label-dot` 삽입 제거**(점 더 이상 안 씀). 대신 `.snav-item.tier-sig`에 클래스만 부여.
- CSS: `.snav-item.tier-sig .snav-ic{color:var(--gold)}` → Signature 항목(스네일) 아이콘이 항상 골드 듀오톤(프리미엄 강조). 페이지 헤더 `◆ Signature` 배지는 그대로.
- (CSS `.snav-label-dot` 규칙·`.snav-item.tier-sig` 위치 규칙은 정리/대체.)

### 4.4 프리미엄 플랫 듀오톤 아이콘 시스템

- **그리드**: `viewBox="0 0 24 24"`, 내용은 2~22 안쪽(동일 옵티컬 사이즈·여백).
- **플랫(면) 듀오톤**: 각 아이콘 = 2개 레이어.
  - 베이스 레이어: `fill="currentColor" fill-opacity=".4"` (덩어리감).
  - 핵심 레이어: `fill="currentColor"` (정의 요소, 풀 불투명).
  - 둘 다 `currentColor` → 항목 색 상속(평소 muted, 활성/Signature 시 골드). 단색 유지로 **통일감**, 듀오톤 깊이로 **고급감**, 면 기반으로 **플랫**.
- **라운드**: 모서리·끝 라운드(`rx`/원형)로 부드러운 프리미엄 톤. 전 아이콘 동일 라운드 규칙.
- **stroke 미사용**(라인 아님) — 면으로만. 전부 같은 제작 규칙.
- **아이콘 매핑(메타포)** — 정확한 path는 구현 계획서에 명시:

  | 항목 | 메타포 |
  |---|---|
  | 이더리움 대시보드 | 4분할 그리드(대시보드) |
  | 계절성 | 월 그리드/달력 |
  | 사이클 오버레이 | 순환(겹친 곡선/원호) |
  | 반감기 | 반분 원(절반 채움) |
  | 스네일 차트 | 나선(소용돌이) |
  | 로그 밴드 | 밴드 채널(상하 띠) |
  | 200주 배수 | 곱셈 ×(겹친 마름모) |
  | 드로다운 | 하향 면적(내림 영역) |
  | 변동성 | 파형(지그재그/맥동) |

### 4.5 모바일

- 가로 스크롤 pill 바에도 아이콘 표시(통일감) — `[아이콘] 라벨` pill. 현재값 배지는 모바일에서 기존대로 숨김.
- 업그레이드 버튼은 헤더 우측 클러스터에서 함께 wrap(≤560px 축소).

## 5. 데이터/로직 흐름 (불변)

- 데이터 로더·`scoreMom/Liq/Fun/Val`·`recompute()`·뷰 라우터(`showView`/`VIEW_DRAW`)·`CHART_TIER` 분류 변경 없음.
- `applyTierBadges()`는 점 삽입 → 클래스 부여로 축소(아이콘 골드는 CSS가 처리). 멱등 유지.
- 업그레이드 버튼은 `showToast`만 호출(기존 유틸 재사용).

## 6. 엣지/주의

- 디자인 토큰만, `html{zoom:1.35}` 유지, 한국어 UI, 좌측 컬러바 금지(아이콘은 면, 좌측 바 아님).
- 아이콘 듀오톤은 `currentColor`라 활성·Signature·hover 색 전환이 자동 — 별도 상태 색 하드코딩 금지.
- `.snav-item` 좌측 묶음(아이콘+라벨)이 길어도 배지가 줄바꿈/겹치지 않게 `min-width:0`·`white-space` 관리.
- 모바일 pill에 아이콘 추가로 폭이 늘어나는지 확인(가로 스크롤이므로 허용, 단 과도하지 않게 17px).

## 7. 검증

- 데스크톱: 9개 사이드 항목에 듀오톤 아이콘 렌더, 통일된 옵티컬 사이즈, 활성 항목 골드, **스네일 아이콘 골드**(점 없음), 배지·그룹 정상, 업그레이드 버튼 골드+클릭 토스트.
- 모바일(≤560px): pill에 아이콘+라벨, 업그레이드 버튼 wrap, 헤더 슬림 유지.
- 회귀: 헤드리스 JS 에러 0, 차트·게이지 정상, 뷰 전환 정상, `applyTierBadges()` 멱등.
- 헤드리스 스크린샷으로 아이콘 고급/통일감 육안 확인(필요 시 path 미세 조정).
