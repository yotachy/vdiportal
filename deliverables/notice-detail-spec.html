5데이터 항목 (공지 상세)

시안은 공지 12건의 상세 데이터를 화면 내 객체(DETAILS)로 보유하며, 목록 화면(notice.html)의 12건과 ID·제목·작성자·등록일이 동일합니다. 날짜는 2026년 3~5월(최신 2026.05.29) 기준 더미 데이터입니다.

6상태 · 분류 정의 (공지 분류)

공지 분류는 중요 · 일반 2종이며, 목록 화면(notice.html)·메인 포탈 관리자 안내 영역과 동일한 분류 체계(cat 값)를 공유합니다.

분류cat 키성격 · 예시상세 화면 표기
중요important점검·보안·장애 복구 등 전사 영향 공지 (예: 정기점검 안내, 랜섬웨어 주의보, 분기 비밀번호 변경)별도 태그 없이 제목 머리말([정기점검]·[보안]·[복구완료])로 성격 표기. cat 값은 목록·포탈 정합용으로 데이터에 보유
일반normal배포·도입·현황 공유 등 정보성 공지 (예: 클라이언트 업데이트, 매뉴얼 배포, 사용 현황 리포트)좌동
상세 데이터의 cat·views·제목·등록일은 목록 화면(notice.html)의 동일 ID 항목과 1:1 정합을 유지해야 합니다. 분류 체계를 변경할 경우 목록·상세·메인 포탈 안내 영역을 함께 갱신합니다.

8연관 화면
  • 공지사항 목록 (notice.html) — 행 클릭 시 본 화면(?id={공지 ID}) 진입 · [목록으로]/breadcrumb로 복귀
  • 메인 포탈 (portal.html) — 하단 관리자 안내 영역 클릭 시 지정된 공지 상세로 직접 진입. [포탈 공지로 지정]으로 노출 공지 변경(관리자)
  • 공지사항 상세 (notice-detail.html) — 이전/다음 글 이동으로 동일 화면 내 공지 전환
  • 로그인 (login.html) — 세션 만료/로그아웃 시
9비고
본 문서의 대상 화면(notice-detail.html)은 UI 시안(프로토타입)입니다. 구현 시 화면 내 더미 데이터(DETAILS)는 공지 상세 조회 API(공지 ID 기반) 응답으로 대체하고, 다음을 함께 처리합니다 — ① 상세 조회 시 조회수 증가 반영(시안은 고정값 보유·미표기) ② 존재하지 않는 공지 ID 요청 시 처리 정책(시안은 최신 공지로 대체) ③ 공지에 첨부파일이 있는 경우 첨부 표시·다운로드 제공(시안에는 첨부 영역 없음 — 상세 API 설계 시 포함 검토) ④ [포탈 공지로 지정]은 관리자 권한 검증 후 실제 지정 처리로 대체.
헤더·사이드바·토스트는 전 화면 공통 영역으로, 별도 공통 컴포넌트로 관리합니다(상세는 공통 정의 참조).
