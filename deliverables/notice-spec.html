5데이터 항목 (공지)

시안은 페이지 내 mock 배열 NOTICES(총 12건, 최신순)로 동작합니다. 기본 기간(최근 1개월) 적용 시 6건이 표시되고, "전체 기간" 선택 시 12건(2페이지)이 표시됩니다.

6상태 · 분류 정의

공지 분류는 중요 / 일반 2종으로 고정합니다. 분류는 데이터 필드 cat으로 관리하며, 목록 화면에서는 별도 분류 태그를 표기하지 않고 고정(핀)·NEW 표식으로 우선순위를 표현합니다. 포탈 메인의 관리자 안내 영역 등 타 화면과 분류 체계를 공유합니다.

6.1 · 분류 (2종)
분류데이터 값성격 · 예시
중요cat: 'important'점검·장애·보안 등 사용자 영향이 큰 공지. 예: 정기 점검, 랜섬웨어 주의보, 네트워크 장비 교체, 복구완료, 비밀번호 변경
일반cat: 'normal'배포·안내성 공지. 예: 클라이언트 배포, 매뉴얼 배포, 보안 패치, 신규 자원 도입, 리포트 공유
6.2 · 목록 표식 규칙
표식데이터 값CSS 클래스표시 규칙 · 색
상단 고정pinned: true.notice-item.pinned · .notice-pin검색·기간 필터 결과 내에서 항상 최상단 정렬. 행 배경 KB옐로우 소프트 그라데이션(var(--kb-yellow-soft)) + 제목 앞 핀 아이콘(var(--kb-yellow-dark)). 시안에서는 1건(정기점검 공지)만 고정
NEW 배지isNew: true.new-badge제목 우측 "NEW". 기본: 빨강 텍스트(var(--danger)) + 연빨강 배경(var(--danger-soft)) / 고정 행: 빨강 배경 + 흰 텍스트 반전
7소스 구조
7.1 · 화면 고유 (notice.html 내부)
  • 고유 스타일 — 목록 그리드(.list-head / .notice-item, 제목 1fr + 작성자 150px + 등록일 110px), 고정 행(.pinned) 그라데이션, 핀 아이콘(.notice-pin), NEW 배지(.new-badge), 820px 이하 작성자 컬럼 숨김
  • 고유 스크립트 — mock 데이터 NOTICES(12건) · 목록 렌더 render()(검색+기간 필터 → 고정 우선 정렬 → 페이지 슬라이스) · 기간 프리셋 setQuickRange() · 검색 입력 리스너 · 핀 아이콘 SVG 상수 · HTML 이스케이프 esc()
7.2 · 공통 의존 (common.css / common.js)
  • common.css — 레이아웃(.top-header/.layout/.sidebar/.main), 페이지 헤더·브레드크럼, 필터 바(.filter-bar/.search-hero/.date-drop), 카드(.fc), 페이저(.pager/.pager-btn), 버튼(.btn/.btn-admin), 관리자 노출 제어(.admin-only + body.admin-mode), 빈 상태(.empty-state)
  • common.js — 헤더/사이드바 주입(data-header/data-sidebar="notice"), 기간 필터 유틸(dateInRange/maxDateOf/quickRangeFrom/DATE_LABELS/toggleDatePop), 페이지네이션(renderPager, PAGE_SIZE=10), 관리자 권한 토글·복원(toggleAdminMode/restoreAdminMode), 인증·이동(goHome/logout), 안내(showToast)
8연관 화면
  • 공지 상세 (notice-detail.html) — 목록 행 클릭 시 ?id={공지 id}로 진입. 상세 측 12건 데이터(id·제목·작성자·등록일)와 정합 유지
  • 공지 등록 (notice-new.html) — 관리자 권한 ON 시 [공지 등록] 버튼으로 진입 (관리자 전용)
  • 메인 포탈 (portal.html) — KPI [최근 공지사항] 카드에서 본 화면으로 진입. 포탈 하단 관리자 안내 영역은 상단 고정 공지(정기점검, notice-detail.html?id=1)와 내용 정합
9비고
본 문서의 대상 화면(notice.html)은 UI 시안(프로토타입)입니다. 시안은 페이지 내 mock 데이터(NOTICES)로 동작하며, 구현 시 공지 목록 API(검색·기간 조건·페이지네이션 서버 처리)로 대체합니다. 조회수(views)는 시안에서 데이터로만 보유하므로 상세 진입 시 집계·노출 방식은 구현 시 별도 정의가 필요합니다. 관리자 권한 토글은 시안용 장치이며, 실제 구현에서는 권한 체계(관리자 역할)에 따라 [공지 등록] 노출을 제어합니다.
헤더·사이드바·토스트는 전 화면 공통 영역으로, 별도 공통 컴포넌트로 관리합니다(상세는 공통 정의 참조).
