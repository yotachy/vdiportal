가상PC(VDI) 접속에 필요한 전용 접속 프로그램의 설치 여부를 확인시키고, 미설치 시 설치 파일 다운로드를 안내한 뒤 메인 포탈로 진입시킴
진입 경로
로그인(login.html) 성공 시 — doLogin()이 인증 플래그 설정 후 본 화면으로 이동 (시안에서는 항상 경유)
사이드바 메뉴
해당 없음 (공통 헤더·사이드바를 사용하지 않는 경유 화면)
전제 조건
로그인 성공 직후 진입 (화면 자체의 별도 인증 검사 없음)

E-03설치 완료 전환다운로드 후 자동 — markInstalled()상태 점 녹색 + 텍스트 "접속 프로그램이 설치되어 있습니다"로 교체 · 다운로드 버튼을 [다시 설치](보조)로 전환 · [포탈 바로가기]를 강조(노랑)로 승격
E-04포탈 바로가기[포탈 바로가기] 클릭 — goPortal()메인 포탈(portal.html)로 이동 (설치 상태와 무관하게 항상 이동 가능)
5소스 구조
5.1 · 화면 고유 스크립트 · 스타일 (install.html 내)
구성 요소설명
.setup-card 외 페이지 스타일헤더·사이드바 없는 화면 중앙 단일 카드 레이아웃(최대 560px). 전역 줌(1.25) 보정을 위해 min-height:calc(100vh / 1.25) 적용. 버튼(.sbtn-primary/.sbtn-outline)·상태 표시(.setup-status)·토스트 스타일 포함
showToast(msg)이 화면 로컬 토스트 함수 (common.js 미사용 화면이므로 자체 정의 — 토스트 요소 없으면 자동 생성, 2.6초 후 사라짐)
downloadInstaller()설치 파일 다운로드 토스트 표시 후 약 0.9초 뒤 markInstalled() 호출 (mock)

goPortal()location.href = 'portal.html' 이동

6연관 화면
  • 로그인 (login.html) — 유일한 진입 원천. 로그인 성공 시 doLogin()이 인증 플래그 설정 후 본 화면으로 이동 (시안에서는 항상 경유. 실서비스에서는 설치되어 있으면 건너뛰고 포탈 직행)
  • 메인 포탈 (portal.html) — [포탈 바로가기] 클릭 시 이동
  • 자료실 (qna.html) — 설치 프로그램 분류에서 동일 설치 파일을 별도 제공 (참고)
7비고
본 문서의 대상 화면(install.html)은 UI 시안(프로토타입)입니다. 시안에서는 설치 여부 판별 없이 항상 미설치 상태로 시작하며, [설치 프로그램 다운로드] 클릭 시 설치 완료 상태로 전환되는 시뮬레이션만 제공합니다.
구현 시에는 접속 클라이언트 설치 감지(설치 여부 자동 판별 — 설치되어 있으면 본 화면을 건너뛰고 포탈 직행)와 설치 파일 배포 시스템 연동(실제 설치 파일 다운로드 — 예: KB-VDI-Setup.exe)으로 대체합니다.
