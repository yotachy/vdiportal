6상태 · 분류 정의

공지 체계의 분류는 두 층위로 구분됩니다. ① 사용자 노출 분류(중요/일반 2종) — 공지사항 목록(notice.html)·상세(notice-detail.html)·포탈 안내 영역의 태그와 동일 체계, ② 등록 화면의 운영 분류(5종) — 운영 관리용으로 사용자 목록에는 표시되지 않습니다.

6.1 · 사용자 노출 분류 (목록 · 상세와 동일 체계)
분류데이터 키표시 클래스(참고)설명
중요important.ntag-important점검·보안·장애 등 전사 영향 공지. 목록에서 강조 태그로 표시
일반normal.ntag-normal배포·매뉴얼·현황 공유 등 일반 안내 공지
6.2 · 등록 화면 운영 분류 (셀렉트 #f_cat)
분류용도
운영 안내 (기본값)일반 운영 관련 안내
시스템 점검정기·비정기 점검, 작업으로 인한 접속 중단 안내
보안보안 공지(랜섬웨어 주의보, 비밀번호 변경 등)
업데이트 · 배포클라이언트·패치·매뉴얼 등 배포 안내
기타그 외 안내
시안의 등록 화면에는 중요/일반 선택 입력이 없습니다(운영 분류 셀렉트만 존재). 사용자 노출 분류(중요/일반)의 지정 방식(운영 분류와의 매핑 또는 별도 선택 항목 추가)은 구현 시 협의 대상입니다. 단, 목록·상세·포탈에 노출되는 최종 분류 체계는 중요/일반 2종으로 고정합니다.
7소스 구조
7.1 · 화면 고유 (notice-new.html 내부)
  • 고유 스타일 — .info-text(작성자 고정 표시 텍스트), .chk-line(포탈 노출 체크박스 라인). 본문 폭 .main { max-width: 1320px }
  • 고유 스크립트 — submitNotice() 단일 함수: 제목 → 본문 순 trim 검증 → 포탈 노출 체크 여부에 따른 토스트 분기 → setTimeout 1.6초 후 notice.html 이동
  • 폼 행 격자(.frow/.frow-label/.frow-field/.frow-help)·입력 컨트롤(.fi/.fs/.fta)·필수 표시(.req)는 공통 스타일 재사용 — 화면 내 재정의 없음
7.2 · 공통 의존 (common.css · common.js) — 관리자 모드 메커니즘 포함
  • 레이아웃 주입 — 헤더([data-header])·사이드바([data-sidebar="notice"])는 common.js가 렌더링. 사용자 정보·메뉴·헬프데스크는 공통 설정(PORTAL_USER·NAV_SECTIONS·SERVICE_DESK)에서 일괄 관리
  • 관리자 모드 — 헤더 [관리자 권한] 버튼 → toggleAdminMode()가 body.admin-mode 클래스를 토글하고 sessionStorage['vdi_admin']에 상태 저장. 페이지 로드 시 restoreAdminMode()가 저장 상태를 복원하여 페이지 이동 간 권한 상태 유지. 로그아웃 시 vdi_admin 함께 제거
  • 관리자 전용 노출 규칙 — .admin-only { display:none !important } / body.admin-mode .admin-only { display:inline-flex !important }. 본 화면의 진입 버튼(공지사항 목록의 [공지 등록])이 이 규칙으로 통제됨. 화면별 추가 처리는 window.onAdminModeChange(on) 훅으로 연결 가능(본 화면은 미사용)
  • 피드백 — 토스트는 공통 showToast(msg) 사용(요소 없으면 자동 생성, 2.8초 후 자동 숨김)
8연관 화면
  • 공지사항 (notice.html) — 진입 화면이자 복귀 화면. 관리자 모드 ON 시 [공지 등록] 버튼 노출, 등록 성공·취소·[공지 목록] 모두 이 화면으로 복귀. 등록된 공지는 목록 최상단에 표시
  • 공지 상세 (notice-detail.html) — 등록된 공지의 사용자 열람 화면 (제목·작성자·등록일·본문 정합)
  • 메인 포탈 (portal.html) — [포탈 노출] 체크 시 첫 화면(내 가상PC) 하단 관리자 안내 영역에 표시 (노출 1건 유지)
  • 공통 헤더 — 관리자 모드 토글([관리자 권한] 버튼)이 위치한 전 화면 공통 영역
9비고
본 문서의 대상 화면(notice-new.html)은 UI 시안(프로토타입)입니다. 구현 시 다음을 실제 로직으로 대체·보강합니다.
① 공지 등록 API — 시안의 등록 처리(토스트 + 목록 이동)는 등록 API 호출·결과 처리로 대체. 포탈 노출 1건 유지(기존 노출 자동 해제) 규칙은 서버에서 보장.
② 권한 체계(관리자 인가) — 시안의 관리자 모드는 화면 데모용 토글(body.admin-mode + sessionStorage)로, 진입 버튼 숨김 수준의 통제입니다. 구현 시 서버측 관리자 인가(메뉴 노출 + 화면 접근 + 등록 API 호출 권한 검사)를 적용하고, 비관리자의 직접 URL 접근을 차단해야 합니다. 작성자(관리자 (VDI 운영팀))는 로그인한 관리자 계정 정보로 대체.
③ 첨부 업로드 정책 — 공지 상세 화면에는 첨부 영역이 있으나 본 등록 시안에는 첨부 입력이 없습니다. 첨부 제공 여부와 허용 확장자·용량·개수 등 업로드 정책은 구현 시 협의하여 확정합니다.
헤더·사이드바·토스트는 전 화면 공통 영역으로, 별도 공통 컴포넌트로 관리합니다(상세는 공통 정의 참조).
