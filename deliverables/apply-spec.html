④ 사유 빈 값(trim) → "신청 사유를 입력해주세요."

6.1 · 페이지 구성 (apply.html 고유)
구분이름역할
페이지 <style>.done-wrap · .done-ring · .done-title · .done-desc · .done-reqno · .done-actions완료 뷰(접수 완료 카드) 전용 스타일 — 중앙 정렬 체크 아이콘 링·제목·설명·신청번호 배지·버튼 묶음
페이지 <script>fmtISO(d)Date → 로컬 기준 YYYY-MM-DD 문자열 변환

페이지 <script>updateStepper(n)stepper의 n단계를 active, 이전 단계를 done, 진행 바(.step-bar)를 fill 처리
페이지 <script>validate()용도·종료일·종료일 범위·사유 순서로 검증, 실패 시 toast 후 false 반환

8비고
본 문서의 대상 화면(apply.html)은 UI 시안(프로토타입)입니다. 승인 요청 제출은 화면 내 뷰 전환으로 시뮬레이션하며, 구현 시 신청 접수 API · 결재선 자동 지정(부서 조직도 연동) · 신청번호 채번이 연동 대상입니다. 신청번호(REQ-2026-0529-017)는 예시 고정값입니다.
본 화면은 본인 신청 전용으로 신청자 정보 입력 영역과 대리 신청 기능이 없으며, VDI 유형은 고정가상화 단일이므로 유형 선택 항목이 없습니다. 헤더·사이드바·토스트는 전 화면 공통 영역으로, 별도 공통 컴포넌트로 관리합니다(상세는 공통 정의 참조).
