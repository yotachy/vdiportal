5데이터 항목 (FAQ 데이터)

FAQ 항목은 페이지 내 FAQS 배열로 관리하며(시안 기준 총 11건), 항목당 필드는 아래 3개입니다.

필드항목설명예시
cat카테고리분류 키 5종 (access / apply / spec / trouble / etc). 화면에는 한글 라벨로 표시access → 접속·로그인
q질문질문 제목 텍스트가상PC에 접속이 되지 않습니다. 어떻게 하나요?
a답변답변 본문(HTML 허용 — 강조·목록·화면 링크)인터넷·사내망 연결 확인 → 일반 리부팅 → 장애신고/헬프데스크(1544-8119) 안내
카테고리별 건수 (시안 기준)
카테고리건수대표 질문
접속·로그인3가상PC 접속 불가 · OTP 미수신 · 비밀번호 분실(5회 실패 시 계정 잠금)
신청·결재3VDI 추가신청 방법 · 결재 진행 상태 확인 · 신청 취소(회수)

6상태 · 분류 정의 (카테고리 체계)

FAQ 분류는 5종으로 고정합니다. 필터 탭·항목 라벨·관리 모달의 카테고리 선택이 모두 동일 체계(CAT_LABEL)를 사용합니다.

분류 키표시 라벨적용 범위
access접속·로그인가상PC 접속, 로그인·2차 인증(OTP), 비밀번호·계정 잠금
apply신청·결재VDI 추가신청, 결재 진행 확인, 신청 회수
spec자원·사양자원 증설, 가상PC 유형(고정가상화) 안내
trouble장애·오류화면 지연·끊김, 주변기기 인식 등 사용 중 장애
etc기타헬프데스크 운영 안내 등 기타 문의

9비고
본 문서의 대상 화면(faq.html)은 UI 시안(프로토타입)입니다. FAQ 데이터는 화면 내 배열(mock)로 관리되며, 구현 시 FAQ 데이터 관리(관리자 등록·수정·삭제) 연동으로 대체합니다 — 관리자 모달의 추가/수정/삭제 동작이 저장 API와 연결되어야 하며, 답변 본문은 HTML 서식을 허용하므로 저장·출력 시 허용 태그 정책(예: 강조·목록·내부 링크)을 함께 정의해야 합니다.
헤더·사이드바·토스트·관리자 권한 토글은 전 화면 공통 영역으로, 별도 공통 컴포넌트로 관리합니다(상세는 공통 정의 참조).
